  • 分析
    • 父进程的pid值为创建的子进程的pid值，pid1的值为父进程自己的pid的值。pid = pid1+1
    • 子进程的pid的值为0，pid1的值为自己的pid的值。
    • 因此，父进程的pid的值等于子进程的pid1的值，都等于子进程的pid。
    • 多次执行时，输出的顺序有所不同，推测是父进程创建子进程之后，二者并行执行，具体执行顺序和CPU的进程调度策略有关。
