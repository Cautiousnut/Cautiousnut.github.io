1. Git基础

1.1. Git的配置

  • Git自带一个git config工具来配置git，每台设备上只需要配置一次即可，git有三个存放于不同位置的配置文件。
    • /etc/gitconfig：包含系统上的每一个用户以及他们仓库的通用配置。使用带有--system的git config命令可以对此文件进行读写。
    • ~/gitconfig或~/.config/git/config：当前用户的配置文件。使用带有--global的git config命令可以对此文件进行读写。
    • .git/config：当前使用仓库的配置文件，在当前仓库的目录中。
    • 下边的配置会覆盖上边的配置。

    1.5. 打标签

  • Git可以给历史中的一个提交打上标签，以示重要，如利用这个功能来标记发布结点(v1.0等)
  • 列出标签
  • 创建标签
    • 两种标签类型：附注标签，轻量标签
    • 附注标签
      • 是存储在Git数据库中的一个完整对象，可以被校验，包含打标签者的名字、电子邮件地址、日期时间和标签信息

    • 轻量标签
      • 知识一个特定提交的引用，不会保留任何信息
