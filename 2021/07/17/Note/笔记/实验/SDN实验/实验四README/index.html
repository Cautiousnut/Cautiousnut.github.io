打补丁的原因

在命令行执行git diff HEAD origin/HEAD得到打补丁之后的代码和源代码之间的区别。由指令执行结果可得，打补丁之后删除了in_port和lasthop,同时增加了一个检测黑洞的方法

增加了检测黑洞的方法之后，程序对黑洞的检测能力增强，因此修改后的程序相较于之前的程序会检测出更多的黑洞。
