QEMU搭建ARM Linux运行环境

安装环境

  • 虚拟机：ubuntu 20.04.2 LTS
  • Busybox：busybox-1.33.1
  • Linux内核：linux-4.9.1
  • QEMU：QEMU 4.2.1
  • arm-linux-gnueabi-gcc 9.3.0

步骤

安装gcc-arm-linux-gnueabi

sudo apt install gcc-arm-linux-gnueabi

安装qumu

sudo apt install qemu

安装配置busybox

  • 下载busybox源代码

    https://busybox.net/downloads/

  • 编译busybox

    make ARCH=arm CROSS_COMPILE=arm-linux-gnueabi- menuconfig 创建.config配置文件

    make ARCH=arm CROSS_COMPILE=arm-linux-gnueabi- 创建busybox可执行文件

    make ARCH=arm CROSS_COMPILE=arm-linux-gnueabi- install 安装busybox(_install目录)

    详见源码包下的INSTALL文档

    ARCH和CROSS_COMPILE用于指定交叉编译的目标平台架构和编译工具，也可直接在Makefile文件中修改

    ​ CROSS_COMOPILE?=arm-linux-gnueabi-

    ​ ARCH?=arm

  • 在_install目录下添加配置相应的目录

    • 需添加的目录

      etc：存放配置文件

      proc：proc文件系统挂载点

      sys：sys文件系统挂载点

      tmp：tmp文件系统挂载点

      dev：设备文件

      lib：库文件

    • dev目录创建如下节点

    • etc/inittab文件

      ::sysinit:/etc/init.d/rcS

      ::askfirst:/bin/sh

      ::ctrlaltdel:/sbin/reboot

      ::shutdown:/sbin/swapoff -a

      ::shutdown:/bin/umount -a -r

      ::restart:/sbin/init

      tty2::askfirst:/bin/sh

      tty3::askfirst:/bin/sh

      tty4::askfirst:/bin/sh

      参考源码包中的examples/inittab

    • etc/fstab文件

      用于指明文件系统挂载点

      #device mount-point type option dump fsck order

      proc /proc proc defaults 0 0

      temps /tmp proc defaults 0 0

      none /tmp ramfs defaults 0 0

      sysfs /sys sysfs defaults 0 0

      mdev /dev ramfs defaults 0 0

    • etc/init.d/rcS文件

      • rcS文件内容

        mount -a

        echo “/sbin/mdev” > /proc/sys/kernel/hotplug

        /sbin/mdev -s # 根据/etc/mdev.conf中的配置进行生成设备节点

        mount -a
