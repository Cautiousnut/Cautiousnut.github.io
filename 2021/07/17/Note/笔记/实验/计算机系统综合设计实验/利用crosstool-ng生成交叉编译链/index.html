利用crosstool-ng生成交叉编译链

计算机84 朱辉 2183414205

实验目的

  • 利用crosstool-ng作为目标系统生成交叉编译链

实验过程

安装crosstool-ng以及依赖包

  • 下载源代码

  • 安装依赖包

编译crosstool-ng

  • 执行./bootstarp

  • 执行./configure

  • 执行make

  • 执行sudo make install

测试crosstool-ng

  • 执行ct-ng -v

制作编译器

  • ct-ng list-samples查看crosstoll-ng工具支持制作哪些工具模块

  • 选择arm-unknown-linux-gnueabi，执行ct-ng arm-unknown-linux-gnueabi生成对应的配置文件.config

  • 执行ct-ng menuconfig命令进入配置界面进行配置

    • paths and misc options配置
      • local tarballs directory ： ./crosstool/src
      • Prefix directory：./crosstool/target
      • Debug crosstool-NG：Save intermediate steps
      • Number of parallel jobs： 4
    • Target options配置
      • Target Architecture ：arm
      • Emit assembly for CPU：arm920t
    • Toolchain options配置
      • Tuple's vendor string：zh
    • C compiler配置
      • Use TM clone registry：选中
    • 其余采用默认配置

  • 执行ct-ng build开始编译

制作编译工具链问题

    • 错误截图

    • 解决方法

      根据build.log中给出的下载失败的地址，手动下载duma_2_5_15放入crosstool/src目录（之前设置的源文件存放目录）

  • 提示cannot find -lstdc++

    • 错误截图

    • 解决方法

  • 提示Build failed in step (top-level)

    • 错误截图

    • 解决方法

      此时为虚拟机硬盘空间不足，扩大虚拟机硬盘空间到50G左右即可
