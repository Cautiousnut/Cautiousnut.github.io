利用构造的编译链完成上层应用开发

计算机84 朱辉 2183414205

实验目的

  • 使用构造出的编译链完成上层应用的开发，并在Qemu中ARM架构的Linux上运行此程序。

实验过程

编程实现SHA-1算法

  • 对于任意长度的明文，SHA1首先对其进行分组，使得每一组的长度为512位，再将每一组扩展划分为80份，然后对每一组进行四轮运算，得到5个链接变量，最后对这些明文分组反复重复处理得到最终结果。

  • 对于每个明文分组的处理过程如下：

    （1） 将512位的明文分组划分为16个子明文分组，每个子明文分组为32位。不足512位的明文则填充到512位后再进行划分

    （2） 申请5个32位的链接变量，记为A、B、C、D、E。

    （3） 16份子明文分组扩展为80份。

    （4） 80份子明文分组进行4轮运算。

    （5） 链接变量与初始链接变量进行求和运算。

    （6） 链接变量作为下一个明文分组的输入重复进行以上操作。

    （7） 最后，5个链接变量里面的数据就是SHA1摘要。

问题及解决方法

  • 编译时提示.init_array section has zero size;.fini_arrray section has zero size

    • 错误截图

    • 解决方法

      参考github上crosstool-ng的issues #1531，设置编译配置时将CC_GCC_TM_CLONE_REGISTERY选上，即将C compiler中的Use TM clone registry选项选上，之后重新编译工具链即可。

  • 使用在线加密工具得到的结果与本地程序不同

    • 错误截图

    • 解决方法

      将file文件下载到windows下用VScode查看，发现末尾多出一个换行符

      因此在在线工具中文本后加一个换行符即可得到相同的结果
