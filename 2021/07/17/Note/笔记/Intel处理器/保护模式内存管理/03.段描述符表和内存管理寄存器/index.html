1. 段描述符表

段描述符表中存放的是段描述符。段描述符表长度不定，最多可以包含8192个8字节的段描述符。有两种段描述符表：GDT和LDT
![段描述符表](_v_images/20210303170444797_17186.png =716x)

1.1. 全局描述符表GDT

每个系统必须有一个GDT表，系统中所有的程序和任务都可以使用它。GDT表自身不是段，它是线性地址空间中的数据结构。GDT的基地址和长度必须被加载到GDTR中。为了提高处理器性能，GDT表的基地址必须在8字节边界上对齐。GDT长度以字节为单位，GDT表长度和基地址相加即可获得有效地址限制。如果长度值为0，则GDT表只有一个有效字节。因为段描述符是8个字节，所以GDT长度项始终是8的整数倍减1。GDT表长度项加1即为GDT表真实长度。
GDT表的第一个表项不会被使用。指向此区域的段选择器被称为null segement selector。空选择器被用作初始化未使用的段寄存器。如果CS或者SS寄存器被置为空选择器会引发异常，其他段寄存器则不会。
当GDTR寄存器被存储时（使用SGDT指令），一个48位的“伪描述符”被存储在内存中。为避免在用户模式（特权级别3）中出现对齐检查错误，伪描述符应位于奇数字地址（即地址MOD 4等于2）。这导致处理器存储一个对其的字，其后跟着一个对其的双字。用户模式程序通常不存储伪描述符，但是通过这种方式对齐伪描述符可以避免产生对齐检查错误。使用SIDT指令存储IDTR寄存器时，应使用相同的对齐方式。存储LDTR或任务寄存器时（分别使用SLDT或STR指令），伪描述符应位于双字地址（即地址MOD 4等于0）
![伪描述符格式](_v_images/20210303213115956_5906.png =715x)

1.2. 局部描述符表LDT

系统中可以有多个LDT表，例如：每个任务可以有一个LDT表，一些任务也可以共享一个LDT表。
LDT表位于LDT类型的数据段中。每个LDT表都应该有其段选择器和段描述符，LDT表的段描述符必须放在GDT表中，但是可以是GDT表的任何位置。
为了避免访问LDT时进行地址转换，LDT的段选择器、基地址、长度以及访问权限都保存在LDTR寄存器中。

1.3. IA-32e模式下的描述符表

此模式下，一个段描述符表最多包含8192个8字节的段描述符。普通段描述符是8个字节，系统段描述符是16个字节（占用两个表项）。
GDTR和LDTR寄存器都被扩展以容纳64位的基地址。对应的伪描述符为80位。

2. 内存管理寄存器

处理器提供了四个内存管理寄存器，分别为GDTR、LDTR、IDTR、TR。这些内存管理寄存器指定控制分段内存管理的数据结构的装载位置。同时处理器提供专门的指令来操作这些寄存器。
![内存管理寄存器](_v_images/20210303213559544_9754.png =712x)

2.1. GDTR

  • 概述
    GDTR即为Global Descriptor Table Register，全局描述符表寄存器。GDTR用于保存GDT的基地址（保护模式下是32位，IA-32e模式是64位）和表长度（16位）。基地址指定GDT起始位置的线性地址，表界限指定表的大小。
  • 指令
    LGDT和SGDT两个指令分别加载（Load）和存储（Store）GDTR寄存器。处理器通电或复位时，基地址设置为0，表限制设置为0FFFFH。在保护模式下的处理器初始化过程中，必须将新的基址加载到GDTR中。

    2.2. LDTR

  • 概述
    LDTR即为Local Descriptor Table Register，局部描述符表寄存器。LDTR寄存器中保存LDT表的16位段选择器、基地址（保护模式下是32位，IA-32模式下为64位）、段长度和段属性。
  • 指令
    LLDT和SLDT指令分别加载（Load）和存储（Store）LDTR寄存器的段选择器部分。包含LDT的段在GDT中必须有一个段描述符。当LLDT指令在LDTR中加载段选择器时：LDT描述符中的基地址、长度和描述符属性将自动加载到LDTR中。
  • 其他
    当任务切换发生时，LDTR将自动加载新任务的LDT的段选择器和描述符。在将新的LDT信息写入寄存器之前，LDTR的内容不会自动保存。处理器通电或复位时，段选择器和基址设置为默认值0，长度设置为0FFFFH

    2.3. IDTR

  • 概述
    IDTR寄存器保存IDT的基址（保护模式下为32位；IA-32e模式下为64位）和16位表限制。基址指定IDT起始位置的线性地址，表限制指定表中的字节数。
  • 指令
    LIDT和SIDT指令分别加载和存储IDTR寄存器。
  • 其他
    处理器通电或复位时，基址设置为0，表限制设置为0FFFFH。作为处理器初始化过程的一部分，可以更改寄存器中的基址和限制。
