Linux用户与用户组管理

基础知识
  • Linux系统是一个多用户多任务的分时操作系统，任何一个要使用系统资源的用户，都必须首先向系统管理员申请一个账号，然后以这个账号的身份进入系统
  • 用户的账号一方面可以帮助系统管理员对使用系统的用户进行跟踪，并控制他们对系统资源的访问，另一方面也可以帮助用户组织文件，并为用户提供安全性保护
  • 每个用户账号都拥有一个唯一的用户名和各自的口令
  • 用户在登录时键入正确的用户名和口令后，就能够进入系统和自己的主目录
  • 任何需要使用操作系统的用户都需要一个系统账号，账号分为：**管理员账号和普通用户账号**
  • 操作系统中根据UID来判断账户，而不是用户名，id为0即为管理员，可以有多个id为0的账户
  • 系统新建账号时，会根据账号类型，自动分配递增账号的UID和GID（用户身份编码，组编号），也可以自行分配。通常情况下应保证UID和GID唯一且不重复
组的类别
  • Linux中，每个用户必须拥有一个主组。创建账号时，系统会自动创建一个同名组作为主组。用户必须属于一个且只有一个主组。用户可以属于0个或者多个附加组。主组又称为基本组，不可以把用户从基本组中删除，但可以更换基本组。
  • 除了基本组之外，用户所在的其他组都是附加组。用户可以从附加组中删除。用户的附加组可以和主组相同，附加组可以直接删除，可以新增和移除任意个所属用户。用户被删除不会影响附加组。
用户与组的配置文件
  • 用户分为以下三类
    • 管理员：root，具有使用系统所有权限的用户，UID=0。
    • 普通用户：使用系统的权限受限，UID=500-60000。
    • 系统用户：保障系统运行的用户，一般不提供密码登录系统，UID=1-499。
  • 用户组配置文件
    • /etc/group，组及其属性信息
    • 使用cat /etc/group查看此文件（好像只能使用这种方式查看）

      其中的每一行表示系统中的一个用户组
    • 含义解释

      每一行都可以分为四列
      第一列：群组名称
      第二列：群组密码，真正的组密码存储在/etc/gshadow文件中
      第三列：群组的ID，即为GID
      第四列：属于这个组的所有用户的列表，此组可以是主组或者附加组，此处只显示将此组作为附加组的用户

  • 用户组密码配置文件
    • /etc/gshadow，组密码及其相关属性
    • 使用cat /etc/gshadow查看此文件
    • 含义解释

      每一行可以分为四列
      第一列：组名
      第二列：组密码
      第三列：组管理员
      第四列：属于这个组的所有用户的列表，此组可以是主组或者附加组，此处只显示将此组作为附加组的用户

  • 用户密码配置文件
    • /etc/shadow，用户密码及其相关属性
    • 其中的文件是和/etc/passwd中的文件一一对应的
    • 使用cat /etc/shadow查看此文件，只有超级用户才能读取此文件（为了保障密码的安全）
    • 含义解释

      每一行可以分为九列
      第一列：登录用户名
      第二列：密码，通常使用shad512加密，带有！表示该账号不能用来登录
      第三列：从1970年1月1日起计算到现在为止最近一次修改密码的时间
      第四列：密码再过几天可以被修改，0表示随时可以修改
      第五列：密码几天后必须变更，99999表示永不过期
      第六列：密码过期前多久提示用户
      第七列：密码过期多久后账户被锁定（用户没有登录活动但是账号仍然能够保持有效的最长时间），如果为0，密码过期直接失效，不能恢复
      第八列：多少天后账户将失效，（从1970-1-1开始算起）
      第九列：保留条目，暂未使用

  • 用户配置文件
    • /etc/passwd，用户及其属性信息
    • 使用cat /etc/passwd进行查看，任何用户都可以进行查看
    • 含义解释

      每一行可以分为七列
      第一列：登录用户名
      第二列：用户密码，真正的密码存储在/etc/gpasswd中
      第三列：用户ID，UId
      第四列：所属组的组ID,GID
      第五列：备注信息
      第六列：用户的主目录（中），当用户登录到到系统时就是在他的home中。是用户登录到系统之后所处的目录，主目录的名称即为用户的登录名,每个用户对自己的主目录有读写和执行的权限。
      第七列：shell类型。在用户登录后，需要启动一个进程，负责将用户的操作传递给内核，这个进程是用户登录到系统之后运行的命令解释器或者 某个特定的程序，即Shell，默认为/bin/sh

    • 伪用户
      • 在/etc/passwd中占有一条记录但是但是不能用于登录，因为他们的登录shell为空
      • 是为了方便系统管理，满足相应系统的进程对文件属主的要求
      • 常见的伪用户定义

        bin：拥有可执行的用户命令文件
        sys：拥有系统文件
        adm：拥有账户文件
        uucp：UUCP使用
        lp：lp或者lpd子系统使用
        nobody：NFS使用

Linux系统用户账号的管理
  • useradd
    • 添加新的用户账号,就是在/etc/passwd中为新用户增加一条信息，同时更新其他相关系统文件
    • 语法：useradd 选项 用户名
    • 参数说明
      • **-c**：指定一段注释性描述 -c comment
      • **-d**：指定用户主目录，如果不存在，同时使用-m选项，创建主目录 -d 目录 -m
      • **-g**：指定用户所属的基本组 -g 用户组
      • **-G**：指定用户所属的附加组 -G 用户组，用户组
      • **-s**：指定用户的登录shell -s shell文件
      • **-u**：指定用户的用户号，如果同时有-o选项，则可以重复使用其他用户的标识号 -u 用户号 -o
    • 各种参数可以配合使用，使用效果是单个效果的叠加。
    • 实例

      useradd -d /home/sam -m sam，添加一个以/home/sam为home的用户账号，账号名称为sam

      useradd -s /bin/sh -g group -G adm,root gem，添加一个gem用户，shell文件为/bin/sh，属于group主组，adm和root附加组
      （组如果不存在则应该先创建组）

  • userdel
    • 删除用户账号，就是将/etc/passwd等系统文件中的该用户信息删除，必要时还需要删除用户的主目录
    • 语法：userdel 选项 用户名
    • 参数说明
      • **-r**：将用户的主目录一起删除
    • 实例

      userdel -r sam，将之前创建的sam用户连同其主目录一同删除

  • usermod
    • 修改账号
    • 语法：usermod 选项 用户名
    • 参数说明
      • 同useradd
      • **-l**：指定一个新的账号，将原来的账号名更改为新的账号名，使用格式：-l 新用户名
      • **-p**：修改账号的口令，使用格式：-p 新口令
    • 实例

      usermod -d /home/sam -m sam，将sam账号的主目录进行更改

  • id
    • 查询账号口令
    • 语法：id 选项 用户名
    • 参数说明
      • **-u**：显示UID
      • **-g**：显示GID
      • **-G**：显示用户所属的组ID
      • **-n**：显示名称
    • 实例
  • passwd
    • 指定和修改用户口令
    • 用户刚刚创建时没有口令，但是被系统锁定，无法使用（在passwd中显示为!，表示无法登陆），必须为其指定口令之后才能使用，即使是空口令
    • 语法：passwd 选项 用户名
    • 参数说明
      • **-l**：锁定口令，禁用账号
      • **-u**：口令解锁
      • **-d**：使账号无口令，即为将口令删除，下一次将无法使用此账号登陆
      • **-f**：强迫用户下次登陆时修改口令
    • 省略用户名则默认为当前用户，超级用户可以修改任意用户的口令，普通用户只能修改自己的口令
    • 直接使用passwd可以修该当前用户的口令，使用passwd 用户名可以在root用户中修改任意账号的口令
Linux系统用户组的管理
  • groupadd
    • 添加新的用户组，用户组的操作实质上是对/etc/group等相关文件的修改
    • 语法：groupadd 选项 用户组
    • 参数说明
      • **-g**：指定新用户的组标识号（GID），默认情况下为递增，使用格式：-g GID
      • **-o**：一般与-g一同使用，表示新用户组的GID可以与系统已有用户组的GID相同
    • 实例

      groupadd a，添加新的用户组a，GID递增
      groupadd -g 1001 -o b，添加新的用户组b，GID为1001

  • groupdel
    • 删除已有的用户组
    • 语法：groupdel 用户组
  • groupmod
    • 修改用户组的属性
    • 语法：groupmod 选项 用户组
    • 参数说明
      • **-g**：为用户指定新的组标识号GID，使用格式：-g GID
      • **-o**：与-g同时使用，新GID可以与已有的GID相同
      • **-n**：将用户组的名字改为新名字，使用格式：-n 新名字
    • 实例
  • newgrp
    • 如果一个用户同时属于多个用户组，那么该用户可以在用户组之间进行切换，以便具有其他用户组的权限
    • 语法：newgrp 用户组
    • 注意：切换到的用户组必须是当前用户的主组或者附加组
  • groupmems
    • 用于更改和查看组中的成员
    • 语法：groupmems 选项 动作
    • 参数说明
      • 动作
      • **-g**：更改为指定组，只有root用户可以使用,使用格式：-g 用户组
      • 选项
      • **-a**：指定用户加入组，使用格式：-a 用户
      • **-d**：从组中删除用户，使用格式：-d 用户
      • **-p**：从组中清除所有成员
      • **-l**：显示组成员列表
    • 实例 显示adm组所有的成员
  • 添加批量用户
    • 用于快速创建大量用户
    • 具体步骤
      • 编辑一个文本用户文件，按照/etc/passwd中的格式书写，注意每个用户的用户名，UID，宿主目录都不相同，密码栏可以留空或者输入*


        此时系统会自动生成用户的密码(/etc/shadow)
      • 以root身份执行/usr/sbin/newusers，从刚创建的文件中导入数据，添加用户
      • 以root身份执行/use/sbin/pwunconv，此命令将/etc/shadow中的密码解码，写回到/etc/passwd中，并且将/etc/shadow的shadow密码栏删掉
      • 按照格式用户名 密码编辑用户的密码对照文件
      • 以root身份运行usr/sbin/chpasswd，此指令用于创建用户密码，会将经过/usr/sbin/passwd命令编码过的密码写入/etc/passwd的密码栏
      • 执行命令/usr/sbin/pwconv将密码编码为shadow passwd，并将结果写入/etc/shadow
