1. Linux文件锁

1.1. 概述

  • Linux中的文件锁技术主要分为劝告锁（advisory lock）和强制锁（mandatory lock）。
  • 强制锁又有两个变种为共享模式强制锁（share-mode mandatory lock）和租界锁（lease）
  • 文件锁有两个类型，分别为共享锁（读锁）和排他锁（写锁）
  • 注意
    • 劝告锁、强制锁等是文件锁的不同技术的实现，是根据锁的内核行为进行的分类。
    • 共享锁、排他锁等是文件锁的不同类型，具有不同的读写性质。

1.2. 共享锁和排他锁

  • 共享锁
    • 又称读锁，S锁。用于不更改或不更新数据的操作（只读操作）。
    • 如果一个进程给文件A加上共享锁，则其他进程只能对A再加共享锁，不能加排他锁。多个共享锁之间不会相互干扰。
    • 给文件加共享锁的进程只能读数据，不能修改数据
  • 排他锁
    • 又称为写锁，X锁。用于数据修改操作（写操作）。
    • 如果一个进程给文件A加上排他锁，则其他进程不能再对A加任何类型的锁，直至排他锁被释放。即当一个进程修改数据时，不允许其他进程同时获取和修改数据。
    • 给文件加排他锁的进程既能读数据，又能修改数据。
  • 二者的存在保证一个文件可以同时拥有多个读者，但是在某一特定时刻，只能拥有一个写者。由此实现进程的同步。

1.3. 劝告锁

  • 是一种协同工作的锁。对于此种锁，内核只提供加锁以及检测文件是否已经加锁的手段，当时内核并不参与锁的控制与协调。
  • 劝告锁是依靠进程自愿遵守的。如果有进程没有检测文件是否已经被其他进程加了锁就直接对其进行操作，内核时不会阻拦的。
  • 使用劝告锁实现同步，进程需要事先对锁的状态做出约定，并根据锁的状态和相互关系确定进程能都对文件执行指定的操作。
  • 劝告锁可以对文件的任意一个部分加锁，也可对整个文件进行加锁，甚至可以对文件将来增大的部分进行加锁。因此一个进程可以获得关于某个文件的不同部分的多个锁。

    1.4. 强制锁

  • 是内核强制采用的一种文件锁。
  • 每当有系统调用open()、read()以及write()时，内核都要检查并确保这些调用不会违反文件上的强制锁。
  • 内核操作
    • 如果一个文件被加上了共享锁，其他进程对此文件进行写操作就会被内核阻止。
    • 如果一个文件被加上了排他锁，其他进程对此文件进行读取或者写操作就会被内核阻止。
  • 进程对加了强制锁的文件进行操作时的行为
    当前锁类型 阻塞读 阻塞写 非阻塞读 非阻塞写
    读锁 正常读取数据 阻塞 正常读取数据 EAGAIN
    写锁 阻塞 阻塞 EAGAIN EAGAIN

1.7. 相关系统调用及函数

  • flock()
    • 函数原型：int flock(int fd,int operation)
    • 参数
      • fd：表示文件描述符
      • operation：表示要进行的锁操作。取值如下
        • LOCK_SH：表示创建一个共享锁。任意时间内，一个文件的共享锁可以被多个进程所拥有
        • LOCK_EX：表示创建一个排他锁。任意时间内，一个文件的排他锁只能被一个进程锁拥有
        • LOCK_UN：表示删除该进程创建的锁
        • LOCK_MAND：
    • 通常情况下，如果加锁请求不能被立即满足，那么此系统调用会阻塞当前进程。
