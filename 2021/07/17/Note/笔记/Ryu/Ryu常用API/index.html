Ryu常用API

1. Ryu应用程序API

  • Ryu应用编程模型

    • Ryu应用是Ryu中实现各种功能的单线程实体。
    • Ryu事件是Ryu应用之间异步传递的信息，也可能来自Openflow控制器。事件可以包含任意的Python对象。
    • 每个Ryu应用维护一个事件的接收队列，采用FIFO调度方式。

  • 创建Ryu应用

    • Ryu应用是ryu.base.app_manager.RyuApp类的子类。
    • Ryu应用只支持单独的实例。

  • 监听事件

    • Ryu应用使用ryu.controller.handler.set_ev_cls修饰器监听特定的事件。

    • ryu.controller.handler.set_ev_cls(ev_cls,dispatcher=None)

      • ev_cls是一个事件类的实例，表示希望接收的事件,也可以为事件类实例的列表。

      • dispatcher说明函数被调用时交换机应该具有的状态。也可以是列表。

        交换机状态 描述
        ryu.controller.handler.HANDSHARK_DISPATCHER 发送并且等待hello信息
        ryu.controller.handler.CONFIG_DISPATCHER 已经进行版本协商和功能请求
        ryu.controller.handler.MAIN_DISPATCHER 接收到交换机属性信息，并且发送配置属性信息
        ryu.controller.handler.DEAD_DISPATCHER 断开连接

  • 生成事件

    • Ryu应用可以使用ryu.base.app_manager.RyuApp类中的方法生成事件。

  • 事件类

    • openflow事件类

      • openflow事件类继承于ryu.controller.ofo_event模块中的ryu.controller.ofp_event.EventOFPMsgBase(msg)类。用于描述接收到的由相连的交换机发出的Openflow信息。

      • openflow事件类至少包含以下属性

        属性 描述
        msg 用于描述相应的openflow信息的对象
        msg.datapath 用于描述openflow信息来源的交换机的对象，是ryu.controller.controller.Datapath的实例
        timestamp 信息源交换机生成Datapath实例的时间戳
        msg.reason

Ryu应用基类

  • class ryu.base.app_manager.RyuApp(*_args,**_kwargs)

    • 属性

    • 方法

      方法 说明
      send_request(req) 发送一个同步请求，并且阻塞等待回复。req是一个EventRequestBase类的实例

2. Ryu controller API

  • 用于实现控制器和交换机之间的互联和事件处理
2.1 controller API

  • 用于定义OpenFlow的控制器，用于处理交换机和控制器的连接等事件，同时还可以产生事件和路由事件

  • ryu.controller.controller.Datapath(socket,address)

    • 用于描述和本控制器相连的交换机

    • 实例拥有如下属性

      属性 描述
      id 64位的openflow Datapath ID，仅适用于ryu.controller.handler.MAIN_DISPATCHER状态
      ofproto 为相应的openflow版本给出openflow数据结构的定义的模块
      ofproto_parser 为相应的openflow版本给出openflow wire消息编码和解码器的模块
      ofproto_parser.OFPxxxx(datapath,…) 为给定的交换机生成openflow消息。可以使用Datapath.send_msg方法发送。xxxx是消息名，参数取决于消息。
      set_xid(self,msg) 生成一个openflow XID并且放到msg.xid中
      send_msg(self,msg) 对OpenFlow消息进行排队以发送到相应的交换机。如果msg.xid为None，则在排队之前会自动对该消息调用set_xid。
      send_barrier 将OpenFlow barrier消息排队发送到交换机
2.2 event、ofp_event API

  • 定义事件系统

  • ryu.controller.event.EventBase

    • 所有事件类的基类，可以通过定义其子类定义自己的事件类。

  • ryu.controller.event.EventRequestBase

    • RyuApp.send_request发出的同步请求的基类。

  • ryu.controller.event.EventReplyBase(dst)

    • RyuApp.send_reply发出的同步回复的基类。

  • ryu.controller.ofp_event.EventOFPStateChange(dp)

    • 用于通知交换机状态改变的事件类。当交换机状态发生改变时，一个此类的实例会被发送到控制器。

    • 实例至少拥有如下属性

      属性 描述
      datapath 相应交换机的ryu.controller.controller.Datapath实例
      reason OFPPR_*之一
      port_on 状态发生改变的端口
2.3 dpset API

  • 定义交换机端的事件。

  • ryu.controller.dpset.EventDP(dp,enter_leave)

    • 用于指示交换机连接和断开连接的状态的事件类

    • 对于openflow交换机，控制器可以通过ryu.controller.ofp_event.EventOFPStateChange获取到同样的信息

    • 实例至少拥有如下属性

      属性 描述
      dp 相应交换机的ryu.controller.controller.Datapath实例
      enter 当交换机连接到控制器时为true，当断开连接时为false
      ports port实例的列表

  • ryu.controller.dpset.EventPortAdd(dp,port)

    • 交换机端口状态为ADD时的通知事件，当交换机添加一个新的端口时会触发此事件。

    • 对于openflow交换机，控制器可以通过ryu.controller.ofp_event.EventOFPPortStatus获取到相同的信息

    • 实例至少拥有如下属性

      属性 描述
      dp 相应交换机的ryu.controller.controller.Datapath实例
      port 新添加端口的端口号

  • ryu.controller.dpset.EventPortDelete(dp,port)

    • 交换机端口状态为DELETE时的通知事件，当交换机移除一个端口时会触发此事件。

    • 对于openflow交换机，控制器可以通过ryu.controller.ofp_event.EventOFPPortStatus获取到相同的信息

    • 实例至少拥有如下属性

      属性 描述
      dp 相应交换机的ryu.controller.controller.Datapath实例
      port 移除的端口的端口号

  • ryu.controller.dpset.EventPortModify(dp,new_port)

    • 交换机端口状态为MODIFY时的通知事件，当交换机更改一个端口时会触发此事件。

    • 对于openflow交换机，控制器可以通过ryu.controller.ofp_event.EventOFPPortStatus获取到相同的信息

    • 实例至少拥有如下属性

      属性 描述
      dp 相应交换机的ryu.controller.controller.Datapath实例
      port 移除的端口的端口号

3. Ryu ofproto_parser API

  • openflow协议的数据结构定义和协议解析
3.2 v1_3协议API

  • class ryu.ofproto.ofproto_v1_3_parser.OFPActionOutput(port,max_len=65509,type_=None,len_=None)

    • 返回action实例，此action实例表明交换机发送参数为(max_len,type,len)的数据包到port端口。

    • 参数

      参数 描述
      port 交换机发送报文的端口
      max_len 发送给控制器的数据的最大长度

  • class ryu.ofproto.ofproto_v1_3_parser.OFPActionSetField(field=None,**kwargs)

    • 返回action实例，当报文匹配到相应的规则时，修改此报文的头部
    • 属性同OFPMatch类

  • class ryu.ofproto.ofproto_v1_3_parser.OFPInstructionActions(type_,actions=None,len_=None)

    • 写入、应用、清除action

    • 参数

      参数 描述
      type 下列值
      OFPIT_WRITE_ACTIONS
      OFPIT_APPLY_ACTIONS
      OFPIT_CLEAR_ACTIONS
      actions Openflow action类实例列表

  • class ryu.ofproto.ofproto_v1_3_parser.OFPMatch(type_=None,length=None,_or-dered_fields=None,**kwargs)

    • 流匹配数据结构

    • 参数列表如下

  • class ryu.ofproto.ofproto_v1_3_parser.OFPFlowMod(datapath,cookie=0,cookie_mask=0,table_id=0,command=0,idle_timeout=0,hard_timeout=0,priority=32768,buffer_id=4294967295,out_port=0,out_group=0,flags=0,match=None,instructions=None)

    • 构造FlowMod报文。控制器发送此信息来修改交换机的流表。

    • 返回msg类实例

    • 参数

      参数 描述
      cookie
      cookie_mask
      table_id
      command 用于指定交换机对匹配的流表进行操作
      OFPFC_ADD 增加流表
      OFPFC_MODIFY 修改流表
      OFPFC_MODIFY_STRICT
      OFPFC_DELETE 删除流表
      OFPFC_DELETE_STRICT
      priority 流表条目的优先级
      buffer_id 提供buffered包或者OFP_NO_BUFFER
      match OFPMatch的实例
      instructions OFPInstructions*实例的列表
      idle_timeout 空闲超时(s)，在此时间内如果流表没有匹配到任何报文则将此流表删除,=0表示无限制
      hard_timeout 硬超时(s)，流表下发之后经过指定时间将被无条件删除，=0表示无限制
      out_port FlowMod报文发出的端口
      out_group
      flags

  • class ryu.ofproto.ofproto_v1_3_parser.OFPPacketOut(datapath,buffer_id=None,in_port=None,actions=None,data=None,actions_len=None)

    • 构造PackageOut报文，用于指导交换机对数据包的转发行为

    • 返回msg类实例

    • 属性

      属性 描述
      buffer_id 用于定位缓存在交换机上的数据，没有缓存则为OPF_NO_BUFFER
      in_port 交换机接收到此数据包的端口或者为OFPP_CONTROLLER
      actions action类实例列表，用于指导交换机处理数据，没有此项则此报文会被抛弃
      data 需要交换机转发的报文，没有buffer_id则必须有此项

  • class ryu.ofproto.ofproto_v1_3_parser.OFPSwitchFeatures(datapath,data-path_id=None,n_buffers=None,n_tables=None,auxil-iary_id=None,capabili-ties=None)

    • 属性回复报文。用于交换机对控制器发出的属性请求信息的回应，包含交换机的属性信息。

4. Ryu Packet API

4.1 构造和解析报文

  • class ryu.lib.packet.packet.Packet(data=None,protocols=None,parse_cls=<class’ryu.lib.packet.ethernet.ethernet’>)

    • 用于报文信息的编码和解码

    • 属性

      属性 描述
      data 用于描述待解码的原始数据报的字节数组。解码后，Packet对象是可列举的。列举的值是协议头和协议携带的数据。协议头是packet_base.PacketBase的子类。携带数据为字符数组。当构建报文时，此属性应该省略。

    • 方法

      方法 描述
      add_protocol(proto) 构造报文，用于注册使用的协议，添加协议头部
      get_protocol(proto) 解析报文，返回第一个与指定协议匹配的协议的头部
      get_protocols(proto) 解析报文，返回所有的与指定协议匹配的协议的头部
      serialize() 构造报文，编码报文，将结果存储在self.data中
4.2 协议头类

  • ryu.lib.packet.arp.arp(hwtype=1,proto=2048,hlen=6,plen=4,opcode=1,src_mac=’ff:ff:ff:ff:ff:ff ’,src_ip=’0.0.0.0’,dst_mac=’ff:ff:ff:ff:ff:ff ’,dst_ip=’0.0.0.0’)

    • ARP协议头部数据结构

    • 属性

      属性 描述
      opcode 操作码
      1 : ARP Request
      2 : ARP Reply
      src_mac 源MAC地址
      src_ip 源IP地址
      dst_mac 目的MAC地址
      dst_ip 目的IP地址

  • ryu.lib.packet.ethernet.ethernet(dst=’ff:ff:ff:ff:ff:ff ‘,src=’00:00:00:00:00:00’,ethertype=2048)

    • 以太网协议头部数据结构

    • 属性

      属性 描述
      dst 源MAC地址
      src 目的MAC地址
      ethertype 协议类型(版本编号？)

  • ryu.lib.packet.ipv4.ipv4(version=4,header_length=5,tos=0,total_length=0,identi-fication=0,flags=0,offset=0,ttl=255,proto=0,csum=0,src=’10.0.0.1’,dst=’10.0.0.2’,option=None)

    • IPv4协议报文头部
