1. Verilog语句

1.1. 赋值语句

1.1.1. 非阻塞赋值方式

  • 语法：a<=b;
  • 在语句块中，上面的语句所赋的变量不能立即为下面的语句所用。
  • 块结束后才能完成这次赋值操作，而所赋的变量值是上一次赋值得到的。
  • 在编写可综合的时序逻辑模块时，这是最常用的赋值方法。
  • 举例

    1.1.2. 阻塞赋值方式

  • 语法：b=a;
  • 赋值语句执行完后，块才结束。
  • 赋值语句结束之后变量的值立即改变。
  • 在时序逻辑中使用时，可能会产生意想不到的结果。
  • 举例

1.4. 块语句

1.4.1. 顺序块

  • 也称为过程块，由关键字begin-end组成
  • 特点
    • 语句顺序执行（除了带有内嵌延迟控制的非阻塞赋值语句）
    • 如果语句包含延迟或者事件控制，那么延迟总是相对于前面那条语句执行完成的仿真时间的。

      1.4.2. 并行块

  • 由关键字fork-join声明
  • 特点
    • 语句并发执行
    • 语句执行顺序由各自语句内延迟或者事件控制决定
    • 语句中的事件控制或者延迟是相对于块语句开始执行的时刻而言的
  • 注意
    • 如果两条语句在同一时刻对同一变量产生影响，那么会引起隐含的竞争，需要尽量避免

      1.4.3. 块语句的特点

  • 嵌套块
    • 块可以嵌套使用，顺序块和并行块可以混合使用
  • 命名块
    • 快可以拥有自己的名字，称为命名块
    • 语法格式
    • 特点
      • 命名块中可以声明局部变量
      • 命名块是设计层次的一部分，命名块中声明的变量可以通过层次名引用进行访问
      • 命名块可以被禁用

        1.4.4. 命名块的禁用

  • 通过关键字disable终止命名块执行，还可用于从循环中退出、处理错误条件以及根据控制信号控制某些代码段是否被执行
  • 类似于带标签的break

1.5. 生成语句

1.5.1. 概述

  • 可以动态的生成Verilog代码，方便了参数化模块的生成
  • 作用
    • 控制变量的声明、任务或者函数的调用，还能对实例引用进行全面的控制
  • 应用场景
    • 对矢量中的多个位进行重复操作
    • 进行多个模块的实例引用的重复操作
    • 根据参数的定义确定程序中是否应该包括某段Verilog代码
  • 生成的实例的类型
    • 模块
    • 用户定义原语
    • 门级原语
    • 连续赋值语句
    • initial和always块
  • 可以在生成范围内声明的数据类型
    • net(线网)，reg(寄存器)
    • integer(整型数)，real(实型数)，time(时间型)，realtime(实数时间型)
    • event(事件)
  • 特点
    • 生成的声明和生成的实例能够在设计中被多次的有条件的调用。
    • 生成的实例具有唯一的标识名，可以使用层次命名规则引用。
    • 生成的数据类型具有唯一的标识名，可以被层次引用
    • 任务和函数的声明也允许出现在生成范围之中，但是不能出现在循环生成当中，具有唯一的标识名，可以层次引用
  • 注意
    • 编写代码时必须在模块中说明生成的实例范围，使用关键字generate-endgenerate指定范围
  • 三种创建生成语句的方法
    • 循环生成
    • 条件生成
    • case生成

      1.5.2. 循环生成语句

  • 允许进行多次引用的模块或者模块项
    • 变量声明
    • 模块
    • 用户定义原语，门级原语
    • 连续赋值语句
    • initial和always块
  • 实例一 说明
    • 仿真开始之前，仿真器会将生成快转换为展开的代码，然后进行仿真。生成快的本质是使用循环内的一条语句来代替多条重复的语句，简化编程
    • genvar用于声明生成变量，生成变量只能用于生成块中，展开后的仿真代码中不存在生成变量
    • 生成变量的值只能由循环生成语句来改变
    • 使用不同生成变量做索引的循环生成语句可以嵌套
    • xor_loop是循环生成语句的名字，通过他可以对循环生成语句中得变量进行层次化访问，各个异或门的相对层次名为：xor_loop[0].g1,xor_loop[1].g1,……

      1.6.2. always语句

  • 在仿真过程中是不断活动的

  • 必须和一定的时序控制结合在一起使用，否则易产生死循环

  • 语法格式

  • 举例

  • always块的触发机制

    • OR事件控制
      • 表示多个事件或者信号中只要一个发生变化都能触发语句或者语句块的执行
      • 举例
      • or也可以使用,来代替
      • 如果敏感变量列表过于繁琐，可以使用@*，@(*)，表示把块的所有输入变量自动包括进敏感列表
    • 电平敏感时序控制
      • 此时语句和语句块要等待某个条件为真时才能执行
      • 举例

  • 注意

    • 一个模块汇总可以有多个always块，他们是并行运行的
    • posedge表示信号上升沿，negedge表示信号下降沿
    • 同一变量不能在不同的always块中进行赋值，容易引起冲突和错误

      1.6.3. task说明语句

  • task说明语句用于定义任务，和函数一起使用可以把一个很大的程序模块分解为许多较小的任务和函数便于理解和调试。

  • 特性

    • 任务完成之后，控制会传回启动过程，如果任务内部有定时控制，则启动的时间可以与控制返回的时间不同。
    • 任务可以启动其他任务，其他任务又可以启动别的任务，可以启动的任务数没有限制。
    • 只有当所有的启动任务完成之后控制才能返回。
    • 输入输出和总线信号的值可以传入、传出任务和函数。

  • 语法

    1.6.4. function说明语句

  • function说明语句用于定义函数

  • 函数的目的是返回一个用于表达式的值

  • 定义函数的语法

  • 调用函数的语法

  • 函数的使用规则

    • 函数的定义不能包含任何时间控制语句，#，@，wait等
    • 函数不能启动任务
    • 定义函数时至少要有一个输入参量
    • 在函数的定义中必须由一条赋值语句给函数中的一个内部变量赋以函数的结果值，该内部变量具有和函数相同的名字

  • 自动递归函数

    • Verilog中的函数无法进行递归调用，如果函数在声明时使用关键字automatic，那么函数就成为可递归的函数，仿真器会为每一次函数调用动态分配新的地址空间。
    • 语法
    • 注意
      • 自动函数中声明的局部变量不能通过层次名进行访问，但是自动函数本身可以通过层次名进行访问

  • 常量函数

    • 是指在仿真开始前的编译阶段计算出的结果为常量的函数，只允许操作常量，不允许访问全局变量或者调用系统函数。
    • 实例

  • 带符号函数

    • 返回值为带符号数的函数

  • function语句和task语句的区别

    • 函数只能与主模块共用一个仿真时间单位，而任务可以定义自己的仿真时间单位。
    • 函数不能启动任务，而任务可以启动其他任务和函数。
    • 函数至少有一个输入变量，而任务可以没有或者有多个任何类型的变量。
    • 函数返回一个值，而任务不返回值。
