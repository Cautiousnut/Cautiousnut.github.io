1. 命名空间

  • 命名空间是为了防止名字冲突而设置的机制，分割了全局命名空间，每个命名空间就是一个单独的作用域。

    1.1. 命名空间定义

    1.1.1. 基础

  • 语法格式
  • 注意
    • 命名空间的名字必须在作用域内保持唯一。
    • 命名空间既可以定义在全局作用域内，也可以定义在其他命名空间中，但是一定不能定义在函数或者类的内部。
    • 每个命名空间都是一个独立的作用域，定义在命名空间中的名字可以被该空间内的其他成员直接访问，也可以被这些成员内嵌作用域中的任何成员访问
    • 在当前命名空间访问另一个独立的命名空间的成员时，需使用::操作符，namespace::name。
    • 命名空间可以不连续，namespcase name{}可以是定义新的命名空间，也可以是向已有的命名空间中添加新的成员。
      • 利用此特性可以将命名空间的类和对象的声明放在头文件中作为接口，将定义部分放在另外的源文件（需要include声明的头文件）中，将接口和实现分离。
      • 通常不能把#include放在命名空间内部，因为这样意味着把头文件中的所有成员定义为该命名空间的成员。
    • 命名空间的成员可以在命名空间内部声明，而在外部进行定义，定义时需使用前缀明确指出其所属的命名空间。此处的外部是指所属命名空间的外层空间，如上一层空间或者全局作用域等，而不是一个毫不相关的命名空间。

      1.1.2. 全局命名空间

  • 在所有类、函数、命名空间之外的空间即为全局命名空间
  • 隐式的声明，没有名字，存在于所有的程序
  • 作用域运算符可以作用于全局命名空间：::name。

    1.1.3. 嵌套命名空间

  • 定义在其他命名空间中的命名空间
  • 嵌套的命名空间也是嵌套的作用域，嵌套在外层命名空间的作用域中。
  • 内层命名空间声明的成员将隐藏外层命名空间中声明的同名成员。
  • 外层空间想要访问内层空间的成员需要添加限定符

    1.1.4. 内联命名空间

  • 内联命名空间是特殊的嵌套命名空间，内联命名空间中的名字可以被外层命名空间直接使用而不用添加任何前缀
  • 语法格式

    1.1.5. 未命名的命名空间

  • 语法格式
  • 未命名的命名空间拥有静态生命周期，在第一次使用前创建，到程序结束时销毁
  • 定义在未命名的命名空间中的名字可以直接使用，不能对他们使用作用域运算符
  • 一个未命名的命名空间可以在一个文件中不连续，但是不能跨越多个文件
  • 两个文件中未命名空间之间没有任何关系，二者内部同名的元素代表不同的实体
  • 如果一个头文件定义了未命名的命名空间，那么该命名空间定义的名字将在不同的包含该头文件的文件中对应不同的实体
  • 可以嵌套在其他命名空间中，访问时使用外层命名空间名字限定。
  • 推荐使用未命名的命名空间声明静态全局实体

    1.2. 访问命名空间中的内容

    1.2.1. 概述

  • 使用::作用域操作符：std::cin表示使用命名空间std中的cin
  • 使用using声明
  • 使用命名空间别名
  • 使用using指示

    1.2.2. using声明

  • 用于访问特定命名空间中的内容
  • 语法格式
  • 注意
    • 每一个using声明只能引入命名空间中的一个成员，每个名字都需要独立的using声明。
    • 头文件中不能有using声明
    • 有效范围从声明的地方开始，到声明所在作用域结束为止
    • 在有效范围内，外层作用域中的同名实体将被隐藏
    • using声明语句可以出现在全局作用域、局部作用域、命名空间作用域以及类的作用域中。
    • 在类的作用域中using声明只能指向基类成员。

      1.2.3. 命名空间的别名

  • 用于为命名空间定义一个较短的别名，便于使用
  • 语法格式
  • 注意
    • 不能在命名空间还没定义时就声明别名
    • 也可以为嵌套的命名空间定义别名namespace A = B::C;
    • 一个命名空间可以有好几个别名，他们之间是等价的。

      1.2.4. using指示

  • 将命名空间成员提升到 包含命名空间本身和using指示 的最近作用域
  • 语法格式
  • 注意
    • 使用using指示后，命名空间中的所有名字都可见。
    • using指示可以出现在全局作用域、局部作用域和命名空间作用域中，但是不能出现在类作用域中。
    • 注意作用域提升之后可能与作用域中原有的命名冲突。
    • using指示应该谨慎使用，以免造成命名空间污染和冲突等问题。
