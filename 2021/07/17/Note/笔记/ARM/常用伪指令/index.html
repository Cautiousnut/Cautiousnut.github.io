常用伪指令

概述

  • 伪指令用于告知汇编器根据指定的要求对汇编程序进行汇编。
  • 使用伪指令可以简化编程过程，是程序更加规范、易读、易交流。
  • ARM中的伪指令和X86中的伪指令很相似但有不同。ARM中的有些伪指令可以汇编成一条至多条机器指令，而X86的伪指令无法汇编为真正的机器指令。

段定义伪指令

  • 用于定义一个逻辑段。

  • 语法

  • 属性

    属性 描述
    CODE 定义代码段，默认为READONLY。代码段入口地址必须是一条指令
    DATA 定义数据段，默认为READWRITE。
    READONLY 指定本段为只读段。
    READWRITE 指定本段为可读可写段。
    ALIGN=x 对齐属性。表示该段的起始地址可以被$2^x$整除,且该段的大小也能被$2^x$字节整除。默认x=2。对于代码段`x != 0
    NOINIT 表示数据段未被初始化或者初始化为0。用于指定可读写预留空间。

  • 注意

    • 程序入口段应满足4字节对齐。

LDR伪指令

  • 语法

  • 用于加载一个地址值(label,32bit)或者立即数(expr,32bit)到Rd寄存器

  • 实现过程

    • 如果是立即数且能够存入第二操作数中,则翻译为MOV指令,否则进行如下操作
    • 将地址值或者立即数保存到数据缓冲池中。
    • 使用采用相对偏移寻址的机器指令LDR，将数据读取到指定的寄存器。
    • 要求数据缓冲池到PC的距离小于4K字节，且默认缓冲池在所有代码之后。
    • 当距离超出4K字节时，需使用LTORG伪指令在合适位置显式的定义数据缓冲池。
    • 此时翻译为LDR指令和DCD指令
