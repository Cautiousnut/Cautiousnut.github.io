数据提交方式

  • GET方式
    • 将请求的地址和参数信息拼接在浏览器的地址栏，请求和参数之间使用?隔开，前是请求的地址，后是参数列表。参数列表使用英文下的&分割，参数要有参数名和参数值，使用=连接
    • 数据明文展示在地址栏中，GET方式提交数据不安全
    • GET方式所能提交的数据大小在2KB左右。
  • POST方式
    • 将请求的参数列表存放在请求数据包的实体内容中，提升了数据的安全性
    • 在提交数据时没有数据大小的限制，文件上传和多媒体等建议使用POST

SpringBoot的自动装配

  • SpringBoot特点：约定大于配置，开箱即用，请求的参数如何快速传递给处理请求的方法。

  • 字符串：如果是字符串参数列表，则会自动将请求的url中名称相同的参数的值传递给处理请求的这个方法的对应参数上。

  • 对象：将url地址中的参数名和对象中的属性名进行比较，如果某个属性的名称和这个参数的名称相同，则将这个参数的值自动的赋值给对象的属性。

状态管理

Cookie
  • 临时用来保存服务器端数据的技术，并且保存在浏览器端，敏感信息不建议保存在cookie中
  • 每个应用程序都对应有cookie，在cookie保存有当前cookie的路径，这个路径用于确定当前的cookie属于哪个应用程序
  • 访问请求时浏览器会自动判断当前的cookie路径是否和当前要访问的服务器一致，如果一致则将当前的数据打包放在请求数据包中一同发送给后台服务器。
Session
  • 用于临时保存服务器端数据的技术，并且将session保存在服务器端，可以用于保存(username,uid,avator)这些信息
  • 每一个session自动生成一个session id来唯一的标识当前的session对象。服务器响应数据时，会自动的将session id发送到浏览器端，浏览器接收当前的session id，将其存放到cookie当中。下一次访问服务器时，cookie会被自动提交给服务器（session id）。tomcat
  • 可设置存活事件，用于访问超时的实现。
  • session对象是一个全局对象，在项目中任何位置都可以获取到这个对象。

final关键字

  • 修饰类，则此类不能被继承
  • 修饰属性，则此变量的值不能被修改，称为常量
  • 修饰方法，则此方法不能被重写（方法名一致、参数列表一致、返回值类型一致、访问修饰符大于等于父类中被重写的方法）

拦截和过滤

拦截器

可以根据开发者的设置来添加白名单和黑名单，通过拦截器设置哪些请求可以被放行，哪些请求需要被拦截。所有请求都被先提交给拦截器，由拦截器进行判断。

开发步骤

  • 定义一个拦截器，实现HandlerIntercepter接口。接口无法实例化，需要定义一个具体的类实现此接口

    处理器拦截器：在此接口中定义了拦截请求的路径对应的方法

    prehandle方法：首先将在触发真正的处理方法之前被框架自动调用，执行时机在所有的处理请求方法之前，在此方法中可以获取到请求相关的所有参数。

  • 注册拦截器，实现WebMvcConfigurer接口，将对应的拦截器添加到拦截器列表中

    • 进行拦截器的添加
    • 添加名单：白名单和黑名单

重定向

  • 用于解决页面跳转的问题。302表示重定向，浏览器会自动的解析此消息头，然后获取此消息头中的新的url地址，自动的向新的url地址发送一个请求。
  • 特点
    • 发送了两次请求，HTTP请求的最大特点是一次请求一次连接。两次请求的请求对象不能共享的。
    • 同应用程序和跨应用程序都可以

转发

  • 用于解决跳转问题。解决的是同一个应用程序中组件的跳转问题，不能跨应用程序使用。
  • 特点
    • 一次请求，共享请求和响应对象。
