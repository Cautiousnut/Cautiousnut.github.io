1.1. 概述

  • 使用软件包管理工具，apt rpm yum

1.2. 使用软件包管理工具

1.2.1. apt

  • Advanced Packaging Tool，高级包装工具。Linux系统中用于管理包系统的工具
  • apt apt-get都是与APT进行交互的命令行工具
  • apt工具包含了apt-get apt-cache apt-config中最常用的命令，并且更加结构化，推荐使用
  • 运行方式
    • apt工具定期从服务器上下载软件包列表，存放在本地数据库中。这个列表中会有可用软件及其依赖信息的记录。
    • 用户执行软件安装操作时，首先apt工具会在本地软件包列表数据库中搜索所要安装软件的信息，然后根据信息在相关服务器上下载软件安装。
  • 相关操作
    • 更新软件列表
    • 安装软件
    • 卸载软件
    • 删除不需要的依赖项
    • 查看包列表
    • 升级包
    • 搜索包
    • 查看包信息
    • 查询安装路径

      1.2.2. dpkg

1.2.3. rpm

  • 是RPM软件包管理工具，来源于Red Hat Linux发行版的软件包管理工具，后被广泛使用
  • rpm软件包命名规则
    • name-version-release.arch.rpm或者name-version-arch.src.rpm
    • name：软件名称
    • version：软件的版本号，格式通常为主版本号.次版本号.修正号
    • release：表示包的发行号，还可包含适用的操作系统
    • arch：表示包适用的硬件平台，noarch表示可在任何平台运行
    • rpm,src.rpm：类型后缀
    • devel：表示此包时软件的开发包
  • 语法
  • 选项
    • -a：查询所有套件
    • -b<完成阶段><套件档>+或-t<完成阶段><套件档>+：设置包装套件的完成阶段，并指定套件档的文件名称
    • -c：列出组态配置文件，配合-l使用
    • -d：列出文本文件，配合-l使用
    • -e<套件档>或--erase<套件档>：删除指定套件
    • -f<文件>+：查询拥有指定文件的套件
    • -h或--hash：显示执行进度
    • -i：显示套件相关信息
    • -i<套件档>或--install<套件档>：安装指定的套件档
    • -l：显示套件的文件列表
    • -p<套件档>+：查询指定的RPM套件档
    • -q：使用询问模式，当遇到问题时，会询问用户
    • -R：显示套件的关联性信息
    • -s：显示文件状态，配合-l使用
    • -U<套件档>或--upgrade<套件档>:升级指定的套件档
    • -v：显示指令执行过程
    • -vv：详细显示指令执行过程
  • RPM包默认安装路径
    安装路径 含义
    /etc/ 配置文件安装目录
    /usr/bin/ 可执行命令安装目录
    /usr/lib/ 程序使用函数库保存位置
    /usr/share/doc/ 基本软件使用手册保存位置
    /usr/share/man/ 帮助文件保存位置
  • 安装.rpm软件包
  • 安装.src.rpm软件包
    • 此类软件包是包含源代码的rpm包，安装时需要进行编译
  • 升级rpm包
  • 查询rpm包
    • -q：query，查询包是否安装，rpm -q 包名
    • -qa：all，查询系统中所有已安装的软件包，rpm -qa
    • -qi：information，查询软件包的详细信息，rpm -qi 包名
    • -ql：查询已安装的软件包的文件及其安装路径，rpm -ql 包名
    • -qlp：package，查询未安装的软件包的文件及其默认安装路径，rpm -qlp 包全名
    • -qf：file，查询某个文件属于哪个rpm包，rpm -qf 文件名
    • qR：requires，查询已安装的软件包依赖的其他包，rpm -qR 包名
  • 卸载rpm包

1.2.4. yum

  • yum包管理器基于RPM包管理，能够从指定服务器自动下载RPM软件包并安装，并自动处理软件包的依赖关系，一次安装所有依赖的软件包。解决了rpm软件包之间的依赖性问题。
  • yum是在Fedora和RedHat以及SUSE中的Shell前端软件包管理器
  • yum与rpm之间的区别
    • rpm需要手动解软件包的依赖性问题。yum是rpm的前端程序
    • yum可自动解决依赖关系，对软件包进行分组，根据分组进行安装，并且配置简单，引入仓库概念，支持多个仓库
  • yum仓库
    • 又称为yum源，用于存放所有的现有rpm软件包。当安装软件时，会在仓库中自动查找软件包以及依赖关系，并安装依赖软件。仓库可以是本地的，也可以是网络的
    • 仓库的配置文件在/etc/yum.repos.d目录下

  • yum源配置

  • 命令形式
    • yum [options][command][package]
    • options：选项
    • command：操作名称
    • package：操作对象
  • 安装软件

1. Linux文件基本属性

Linux是一种典型的多用户系统，不同的用户处于不同的地位，拥有不同的权限，Linux系统对不同用户访问同一文件的权限做了不同的规定
Linux文件属主和属组

  • 属主
    • 文件的特定的持有者，对该文件持有所有权的用户
  • 属组
    • 用户按组分配，一个用户属于一个或者多个组
  • 其他用户
    • 属主和属组之外的用户为其他用户

文件属性基本格式
文件类型 属主权限 属组权限 其他用户权限

  • 文件类型
    • Linux中第一个字符代表这个文件的类型，有如下几种可能
    • **d**：表示是目录
    • **-**：表示是文件
    • **l**：表示为链接文档
    • **b**：表示装置文件里面可供存储的接口设备
    • **c**：表示转置文件里边的串行接口设备，如鼠标、键盘等
  • 属主权限
    • 表示文件所有者拥有的该文件的权限
    • 三个参数为一组，为rwx的三个参数的集合
    • **r**：表示可读（read）
    • **w**：表示可写（write）
    • **x**：表示可执行（execute）
    • **-**：如果没有某个权限则使用-代替
  • 属组权限
    • 表示文件所有者的同组用户拥有的该文件的权限
    • 其余同属主
  • 其他用户权限
    • 表示其他用户拥有的该文件的权限
    • 其余同属主
  • 举例

更改文件属性

说明：文件的属性必须由文件的属主或者root用户来修改，其他情况无法进行修改

  • **chgrp**：更改文件属组
    • 语法：chgrp [-R] 属组名 文件名
    • 参数选项
      • **-R**：递归更改文件属性，在更改某个目录文件的属组时，如果加上-R的参数，该目录下的所有文件的属组都会改变
  • **chown**：更改文件的属主和属组
    • 语法
      • chown [-R] 属主名 文件名
      • chown [-R] 属主名：属组名 文件名
    • 举例

      1.将用户切换为root: su root

      2.chown bin:bin a

  • chmod:更改文件的属性
    1.数字类型改变文件权限
    • 语法：chmod [-R] xyz 文件或目录
    • 选项与参数
      • **xyz**：即为数字类型的权限属性，x为属主三个属性对应二进制的值，yz同x，如：rwx=7
      • **-R**：递归变更，次目录下所有文件的属性都会改变
    • 举例

  • 语法
    • 1.更改文件权限：chmod u=权限,g=权限,o=权限 文件名
    • 2.删除文件权限：chmod a-r|w|x 文件名
  • 选项参数
    • **u**：表示属主权限（user）
    • **g**：表示属组权限（group）
    • **o**：表示其他用户权限（others）

1. Linux文件与目录管理

1.1. 一、路径

  • 根目录：/
  • 绝对路径：从根目录 / 写起，如/a/b/c
  • 相对路径：不是由根目录写起，比如从/a/b/c/e要到/a/b/c/d，则写为../d

  • rmdir（删除空的目录）
    • 语法：rmdir [-p] 目录名称

  • cp（复制文件或者目录copy）
    • 语法：
      • 复制一个文件或者文件夹： cp [option] src target
      • 复制多个文件，目标是一个文件夹： cp [option] src1 src2 …… target
    • 常用选项：
      • -r： 递归复制，自动复制文件夹内部的文件夹，自动创建目标位置不存在的文件夹。多用于目录的复制

      • -v： –verbose 详细显示进行的步骤
    • 注意：rmdir只能删除空的目录，而rm命令可以删除任意目录和文件

    • 实例：

  • mv(移动文件与目录，或者修改名称)
    • 语法：mv [option] src destina(可以有多个源文件，中间使用空格隔开)
    • 选项与参数：
      • -f： –force 如果目标文件已经存在，不会询问，直接覆盖
      • -i： 互动模式，如果存在会询问是否覆盖
      • -u： 如果目标文件已经存在，并且源文件较新时才会升级

    • 实例
      mv a d
      将a目录下的所有文件移动到d目录下

1. Linux文件锁

1.1. 概述

  • Linux中的文件锁技术主要分为劝告锁（advisory lock）和强制锁（mandatory lock）。
  • 强制锁又有两个变种为共享模式强制锁（share-mode mandatory lock）和租界锁（lease）
  • 文件锁有两个类型，分别为共享锁（读锁）和排他锁（写锁）
  • 注意
    • 劝告锁、强制锁等是文件锁的不同技术的实现，是根据锁的内核行为进行的分类。
    • 共享锁、排他锁等是文件锁的不同类型，具有不同的读写性质。

1.2. 共享锁和排他锁

  • 共享锁
    • 又称读锁，S锁。用于不更改或不更新数据的操作（只读操作）。
    • 如果一个进程给文件A加上共享锁，则其他进程只能对A再加共享锁，不能加排他锁。多个共享锁之间不会相互干扰。
    • 给文件加共享锁的进程只能读数据，不能修改数据
  • 排他锁
    • 又称为写锁，X锁。用于数据修改操作（写操作）。
    • 如果一个进程给文件A加上排他锁，则其他进程不能再对A加任何类型的锁，直至排他锁被释放。即当一个进程修改数据时，不允许其他进程同时获取和修改数据。
    • 给文件加排他锁的进程既能读数据，又能修改数据。
  • 二者的存在保证一个文件可以同时拥有多个读者，但是在某一特定时刻，只能拥有一个写者。由此实现进程的同步。

1.3. 劝告锁

  • 是一种协同工作的锁。对于此种锁，内核只提供加锁以及检测文件是否已经加锁的手段，当时内核并不参与锁的控制与协调。
  • 劝告锁是依靠进程自愿遵守的。如果有进程没有检测文件是否已经被其他进程加了锁就直接对其进行操作，内核时不会阻拦的。
  • 使用劝告锁实现同步，进程需要事先对锁的状态做出约定，并根据锁的状态和相互关系确定进程能都对文件执行指定的操作。
  • 劝告锁可以对文件的任意一个部分加锁，也可对整个文件进行加锁，甚至可以对文件将来增大的部分进行加锁。因此一个进程可以获得关于某个文件的不同部分的多个锁。

    1.4. 强制锁

  • 是内核强制采用的一种文件锁。
  • 每当有系统调用open()、read()以及write()时，内核都要检查并确保这些调用不会违反文件上的强制锁。
  • 内核操作
    • 如果一个文件被加上了共享锁，其他进程对此文件进行写操作就会被内核阻止。
    • 如果一个文件被加上了排他锁，其他进程对此文件进行读取或者写操作就会被内核阻止。
  • 进程对加了强制锁的文件进行操作时的行为
    当前锁类型 阻塞读 阻塞写 非阻塞读 非阻塞写
    读锁 正常读取数据 阻塞 正常读取数据 EAGAIN
    写锁 阻塞 阻塞 EAGAIN EAGAIN

1. Linux系统目录结构与内核文件

1.1. 系统根目录下的文件目录名及功能

  • 概览
  • /bin
    • Binaries，存放最经常使用的命令文件
  • /boot
    • 存放启动Linux的核心文件，包括一些连接文件以及系统镜像
  • /dev
    • Device，存放Linux的外部设备，提供了访问外部设备的端口，在Linux中所有设备的访问都是文件的形式
  • /etc
    • etcetera，存放系统管理所需配置文件和子目录
  • /home
    • 用户主目录，每个用户有一个自己的目录，一般目录名以自己的账号命名。
  • /lib
    • Library，存放系统最基本的动态连接共享库。
  • /lost+found
    • 一般为空，非法关机后会存放一些文件
  • /media
    • 系统自动识别一些设备，如U盘、光驱等。识别之后，系统会将识别到的设备挂载到此目录下。
  • /mnt
    • 在此目录下用户可以将别的文件系统临时挂载到系统中，然后在该目录下可以查看文件系统的内容
  • /opt
    • optional，给主机额外安装软件所摆放的目录，默认是空的。
  • /proc
    • processes，是一种虚拟文件系统，存储的是当前内核运行状态的一系列特殊文件，是系统内存的映射，是虚拟的目录。可以通过直接访问此目录还获取系统信息
    • 此目录的内容不在硬盘中，而是在内存中，可以直接修改里边的文件。
  • /root
    • 系统管理员，也称超级权限值者的用户主目录
  • /usr
    • Unix Shared resources,存放用户的应用程序和文件
    • /usr/bin
      • 系统用户使用的应用程序
    • /usr/sbin
      • 超级用户使用的高级管理层序和系统维护程序
    • /usr/src
      • 内核源代码默认放置目录
