Flex基础

  • 概述：也称为lex。Lex工具的输入表示方法称为Lex语言，工具本身称为Lex编译器。

    Lex程序的结构

  • 基本形式如下

    定义部分

  • 代码部分
    • 此部分的代码将被直接复制到生成的C文件中
    • 有两种形式：%top{……}和%{……%}
    • %top{……}:其中的代码会被拷贝到C文件的顶部。一般是注释和预编译命令(如#include)
    • %{……%}:常用与定义一些函数声明，结构体，类型等，还可重定义一些Flex的宏
  • 选项部分
    • %option noyywrap：不使用yywrap()
    • %option warn：开启所有警告
    • %option nodefault：去除默认的规则。此时输入无法匹配时会报告错误
  • 状态定义
  • 模式定义
    • 为正则表达式命名，在规则部分可以加上{}后直接调用

      规则部分

    • 格式：模式 {动作}
    • 模式是正则表达式，动作是当其前方的正则表达式被匹配时所执行的动作。
    • 模式必须在行首，否则会被视为代码直接复制到生成的程序中

      用户附加程序部分

    • 辅助函数部分的代码会被直接复制到文件lex.yy.c中。
    • 此部分通常用于定义在转换规则部分的动作中所需要使用的函数。

      Lex的使用方式

  • 工作流程如图
  • 经过上图所示步骤，会得到一个读取输入字符流并生成词法单元流的可运行的词法分析器
  • 处理二义性的基本准则
    • 总是选择最长的前缀
    • 如果最长的前缀与多个模式匹配，总是选择在Lex程序中先被列出的模式

      Lex库以及自动生成的变量、函数

      变量

  • yytext
    • 指向识别到的词素开头的指针。
  • yyleng
    • 存放识别到的词素的长度。
  • yylval
    • 在词法分析器和语法分析器之间共享的整型变量，可用在二者之间传递识别到的词素的附加信息。可以根据需要重新定义。
  • yylloc
    • 匹配的字符串在整个输入流中的位置。
  • yyin
    • 词法分析器默认通过此文件句柄读取输入
  • yyout
    • 默认情况下，所有未被匹配的输入都会拷贝到yyout，

      函数

  • yylex()
    • 由lex程序自动创建的扫描程序的入口点。单独运行Lex程序并且没有链接库文件时，需要显示的在main程序中调用此函数开始词法分析。
    • 每当程序需要一个记号时，会调用yylex()读取输入，当识别出一个记号时，yylex()会返回。当程序需要下一个记号时，yylex()会被再次调用，此时会以后续的输入字符继续分析。如果匹配之后不返回，则词法分析器会在此次调用中继续分析接下来的字符。
  • yywrap()
    • 当词法分析器到达yyin的结束位置时，会调用此函数。当有另一个输入文件时，yywrap()可以调整yyin的值并且返回0来重新开始词法分析。返回1则结束词法分析。
    • Lex库中默认的yywrap()返回1。不链接库时，可以在源程序中显示的定义一个同样的yywrap()（直接返回1）。
    • 可以在源程序开头使用%option noyywrap要求不使用yywrap()
  • yyrestart(f)
    • f为文件句柄
    • 使用此方法可以读取多个文件

      Lex库

  • Flex和Lex总是提供一个小型的库，叫做-lfl。其中定义了默认的main例程和yywrap()

1. 命名空间

  • 命名空间是为了防止名字冲突而设置的机制，分割了全局命名空间，每个命名空间就是一个单独的作用域。

    1.1. 命名空间定义

    1.1.1. 基础

  • 语法格式
  • 注意
    • 命名空间的名字必须在作用域内保持唯一。
    • 命名空间既可以定义在全局作用域内，也可以定义在其他命名空间中，但是一定不能定义在函数或者类的内部。
    • 每个命名空间都是一个独立的作用域，定义在命名空间中的名字可以被该空间内的其他成员直接访问，也可以被这些成员内嵌作用域中的任何成员访问
    • 在当前命名空间访问另一个独立的命名空间的成员时，需使用::操作符，namespace::name。
    • 命名空间可以不连续，namespcase name{}可以是定义新的命名空间，也可以是向已有的命名空间中添加新的成员。
      • 利用此特性可以将命名空间的类和对象的声明放在头文件中作为接口，将定义部分放在另外的源文件（需要include声明的头文件）中，将接口和实现分离。
      • 通常不能把#include放在命名空间内部，因为这样意味着把头文件中的所有成员定义为该命名空间的成员。
    • 命名空间的成员可以在命名空间内部声明，而在外部进行定义，定义时需使用前缀明确指出其所属的命名空间。此处的外部是指所属命名空间的外层空间，如上一层空间或者全局作用域等，而不是一个毫不相关的命名空间。

      1.1.2. 全局命名空间

  • 在所有类、函数、命名空间之外的空间即为全局命名空间
  • 隐式的声明，没有名字，存在于所有的程序
  • 作用域运算符可以作用于全局命名空间：::name。

    1.1.3. 嵌套命名空间

  • 定义在其他命名空间中的命名空间
  • 嵌套的命名空间也是嵌套的作用域，嵌套在外层命名空间的作用域中。
  • 内层命名空间声明的成员将隐藏外层命名空间中声明的同名成员。
  • 外层空间想要访问内层空间的成员需要添加限定符

    1.1.4. 内联命名空间

  • 内联命名空间是特殊的嵌套命名空间，内联命名空间中的名字可以被外层命名空间直接使用而不用添加任何前缀
  • 语法格式

    1.1.5. 未命名的命名空间

  • 语法格式
  • 未命名的命名空间拥有静态生命周期，在第一次使用前创建，到程序结束时销毁
  • 定义在未命名的命名空间中的名字可以直接使用，不能对他们使用作用域运算符
  • 一个未命名的命名空间可以在一个文件中不连续，但是不能跨越多个文件
  • 两个文件中未命名空间之间没有任何关系，二者内部同名的元素代表不同的实体
  • 如果一个头文件定义了未命名的命名空间，那么该命名空间定义的名字将在不同的包含该头文件的文件中对应不同的实体
  • 可以嵌套在其他命名空间中，访问时使用外层命名空间名字限定。
  • 推荐使用未命名的命名空间声明静态全局实体

    1.2. 访问命名空间中的内容

    1.2.1. 概述

  • 使用::作用域操作符：std::cin表示使用命名空间std中的cin
  • 使用using声明
  • 使用命名空间别名
  • 使用using指示

    1.2.2. using声明

  • 用于访问特定命名空间中的内容
  • 语法格式
  • 注意
    • 每一个using声明只能引入命名空间中的一个成员，每个名字都需要独立的using声明。
    • 头文件中不能有using声明
    • 有效范围从声明的地方开始，到声明所在作用域结束为止
    • 在有效范围内，外层作用域中的同名实体将被隐藏
    • using声明语句可以出现在全局作用域、局部作用域、命名空间作用域以及类的作用域中。
    • 在类的作用域中using声明只能指向基类成员。

      1.2.3. 命名空间的别名

  • 用于为命名空间定义一个较短的别名，便于使用
  • 语法格式
  • 注意
    • 不能在命名空间还没定义时就声明别名
    • 也可以为嵌套的命名空间定义别名namespace A = B::C;
    • 一个命名空间可以有好几个别名，他们之间是等价的。

      1.2.4. using指示

  • 将命名空间成员提升到 包含命名空间本身和using指示 的最近作用域
  • 语法格式
  • 注意
    • 使用using指示后，命名空间中的所有名字都可见。
    • using指示可以出现在全局作用域、局部作用域和命名空间作用域中，但是不能出现在类作用域中。
    • 注意作用域提升之后可能与作用域中原有的命名冲突。
    • using指示应该谨慎使用，以免造成命名空间污染和冲突等问题。

1. 顺序容器

1.1. 概述

  • 顺序容器：提供控制元素存储和访问顺序的能力。这种顺序与元素的值无关，只与元素加入容器的顺序相关。

  • 标准库提供的顺序容器

    容器名称 容器特性
    vector 可变大小数组。支持快速随机访问。在尾部之外的位置插入或删除元素可能很慢
    array 固定大小数组。支持快速随机访问。不能添加或者删除元素
    string 专门用于保存字符的可变大小数组。支持快速随机访问。在尾部插入删除速度很快
    deque 双端队列。支持快速随机访问。在头或者尾部插入或者删除速度很快
    list 双向链表。支持双向顺序访问。任何位置进行插入删除速度都很快
    froward_list 单向链表。支持单向顺序访问。任何位置进行插入删除速度都很快

  • 注意

    • 标准库容器的性能几乎肯定与最精心优化过的同类数据结构一样好。所以建议使用标准库容器而不是原始的数据结构。
    • 一般情况下推荐使用vector。

1.1.1. Bochs 是个啥？

简单来说 bochs 是一个模拟器，能够完整的模拟一台计算机。详细来讲 bochs 是 X86 硬件平台的开源模拟器，完全可以通过软件来给我们提供各种所需的硬件资源。

和 bochs 类似的虚拟机软件还有我们常用的 VMware、Virtuabox，但区别也是明显的。bochs 是完全依靠软件来模拟整个环境的：从启动到重启包括 PC 的外设键盘、鼠标、磁盘以及网卡等，全部都是由软件来模拟的，而其余软件则不然 (部分依赖于硬件)。
