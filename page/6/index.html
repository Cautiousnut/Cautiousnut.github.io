1. Icarus Verilog的使用

1.1. 概述

  • Icarus Verilog是一个轻量、免费、开源的Verilog编译器，基于C++实现，可以用于Verilog文件的语法检查以及一些基本的的时序仿真
  • 包含3个工具
    • iverilog：用于编译verilog和VHDL文件，进行语法检查，生成可执行文件
    • vvp：根据可执行文件，生成仿真波形文件
    • gtkwave：用于打开仿真波形文件，图形化显示波形
  • Iverilog常用参数
    • -o：用于指定生成文件的名称，-o test，如果不指定默认为a.out
    • -y：用于指定包含文件夹，当top.v中代用其他.v模块时，如果找不到调用模块就使用此命令显式指定,-y path
    • -tvhdl：把verilog文件转换为VHDL文件，-tvhdl

      1.2. Verilog编译仿真实际过程

  • 开始仿真前要testbench文件产生测试信号，同时注意文件中必须要有以下编译器专用语句，否则无法生成vcd文件
  • 编译
    • iverilog -o wave haedreg_top.v
    • 注意此时编译的应该是testbench文件
  • 生成波形文件
    • vvp -n wave -lxt2
    • 运行之后会在当前目录下生产.vcd文件
  • 打开波形文件
    • gtkwave wave.vcd
  • 转换为VHDL文件
    • iverilog -tvhdl -o a.vhd hardreg_top.v
  • VHDL文件的编译
    • 在原编译命令的基础上添加-g2012即可

编译预处理

  • 编译系统通常对这些特殊的命令进行预处理，然后将预处理的结果和源程序一起进行通常的编译处理

    宏定义`define

  • 用指定的标识符来代表一个字符串
  • 语法格式
  • 说明
    • 宏名可以使用大写字母表示也可以使用小写字母表示，但是建议使用大写字母，和变量名相区别
    • 宏定义指令可以出现在模块定义里边，也可以出现在模块定义外边。有效范围为定义命令之后到源文件结束，通常写在模块定义的外边
    • 在引用已经定义的宏名时，必须在宏名的外边加上符号`,表示该名字是一个经过宏定义的名字。
    • 宏定义是使用宏名替代一个字符串，宏展开时只是做简单的置换，不会进行语法检查
    • 宏定义不是语句，不用在末尾加分号，如果加了会连分号一起替换
    • 在引用宏定义时可以引用已经定义的宏名，可以进行层层置换
    • 宏名和宏内容必须在同一行中进行声明，如果宏内容中包含注释，注释不会作为被置换的内容
    • 宏内容可以是空格，此时宏内容被定义为空，引用此宏名不会有任何内容被替换
  • 注意：
    • 组成宏内容的字符串不能被以下语句记号分隔开
      • 注释行
      • 数字
      • 字符串
      • 确认符
      • 关键字
      • 双目和三目运算符
    • 建议不要使用和预处理命令同名的宏名，有些EDA软件不支持
    • 宏名可以和普通标识符同名，但是容易混淆，不建议使用

      `include

  • 一个源文件将另一个源文件的内容全部包含进来
  • 语法格式：
  • 在编译预处理时，会对此命令包含的文件内容进行复制并插入到预处理命令出现的地方
  • 说明
    • 一个include命令只能指定一个被包含的文件
    • 此命令可以出现在源文件的任何地方，被包含的文件名可以是相对地址也可以是绝对地址
    • 可以将多个include命令写在同一行，用空格隔开
    • 包含文件是可以嵌套的

      `timescale

  • 作用：用于说明跟在该命令后的模块的时间单位和时间精度，可以在同一个设计中包含不同的时间单位的模块
  • 语法格式
  • 说明
    • 如果一个程序中包含多个此命令，则使用最小的时间精度决定仿真的时间单位
    • 时间精度至少要和时间单位一样精确，时间精度值不能大于时间单位值
    • 时间单位和时间精度必须是整数，有效数字为1,10,100，单位为s,ms,us,ns,ps,fs

流水线

  • 将规模较大层次较多的组合逻辑电路分为几个级，在每一集插入寄存器组并暂存中间数据，上一级的输出是下一级的输入
  • 流水线的引入提高的只是系统的吞吐率，即单位时间内执行的指令数目，但是单条指令执行的绝对时间是增加的
  • 流水线是在串行中实现了形式上的并行，是用面积换取速度的思想的体现。
  • 流水线常用于使用频率较高的逻辑电路设计
  • 使用verilog语言描述流水线结构的运算部件时，要是用结构描述，才能够真正综合成设计者想要的流水线结构

  • 用可综合的Verilog模块设计复杂的多输出状态机时常用的方法
    • 在负责状态机的实现过程中常常把状态的变化与输出开关的控制分成两部分来考虑，还常常把每一个输出开关写成一个个独立的always块。

设计可综合状态机的指导原则

概述

  • FPGA内部触发器数量相当多，同时独热码状态机的译码逻辑最为简单，所以在设计采用FPGA实现的状态机时常用独热码状态机
  • 建议采用case相关语句建立状态机的模型，更加清晰明了。
  • 使用case语句时最好使用default设置状态为x，更加符合硬件通电后的真实状态。
  • 状态机应该有一个异步或者同步的复位端，以便在通电时将硬件电路复位到有效状态。
  • 大多数的综合器都不支持在一个always块中由多个事件触发的状态机，描述状态机时应明确的由唯一的时钟触发。如果必须使用不同的时钟触发，则把他们编写为两个模块，然后用实例引用的方式连接起来，为了调试方便，应该尽量使得两个状态机的时钟有一定关系。
  • 异步状态机无法直接综合，如果需要设计异步电路，建议使用电路图输入的方法，或者用引用实例的写法把几个引用的实例用异步时钟连接起来。

    综合的一般原则

  • 综合之前一定要进行仿真，排查逻辑错误。
  • 每一次布局布线之后都必须进行仿真，在器件编程或者留片之前进行最后的仿真。
  • 直接描述的异步状态机无法综合。
  • 如果要为电平敏感的锁存器建模，使用连续赋值语句时最简单的方法。

    语言指导原则

  • always块
    • 每个always块只能有一个事件控制@(event-expression)，而且要紧跟在always关键字之后
    • always块可以表示时序逻辑或者组合逻辑。
    • 带有posedge和negedge关键字的事件表达式表示沿触发的时序逻辑，没有则表示组合逻辑或者电平敏感的锁存器，或者二者都表示。
    • 每个表示时序的always块只能由一个时钟跳变沿触发，置位或者复位最好也由该时钟跳变沿触发。
    • 在always块中赋值的信号必须定义为reg类型或者整型
    • always块中应该避免组合反馈回路。每次执行always块时，在生成组合逻辑的always块中赋值的所有信号必须有明确的值，否则需要设计者在设计中加入电平敏感的锁存器来保持赋值前的最后一个值，只有这样综合器才能正常生成电路。进一步解释：用always块设计纯组合逻辑电路时，在生成组合逻辑的always块中参与赋值的所有信号都必须有明确的值，即在赋值表达式右端参与赋值的信号都必须在always@(敏感电平列表)中列出。如果赋值表达式右端引用了没有列出的信号，在综合时，将会为该没有列出的信号隐含的产生一个透明锁存器。因为此时该信号的变化不会立刻引起所赋值的变化，需要暂存起来，等待always块被触发时发挥作用。这样电路就不是纯组合逻辑电路了，综合器回报警，提示插入了锁存器。

      状态机的置位与复位

      状态机的异步置位与复位
  • 异步置位与复位是与时钟无关的，当异步置位与复位到来时他们立即分别置触发器的输出为1或者0，不需要等待时钟沿
  • 状态机的异步置位与复位是用always块和事件控制实现的。
  • 事件控制语法
  • 实例
    状态机的同步置位与复位
  • 同步置位与复位是指只有在时钟的有效跳变沿时刻置位或者复位，信号才能使触发器置位或者复位，always块的执行应该只由时钟有效跳变沿触发

1. 深入理解阻塞与非阻塞赋值的不同

1.1. 阻塞赋值和非阻塞赋值的异同

1.1.1. 概述

  • 两个要点
    • 在描述组合逻辑的always块中使用阻塞赋值，则综合成组合逻辑的电路结构
    • 在描述时序逻辑的always块中使用非阻塞赋值，则综合成时序逻辑的电路结构
  • 原因
    • 因为要使综合前仿真和综合后仿真一致。如果不按上述方式编写代码，前后仿真的结果可能会不一致
  • 名词
    • RHS：赋值等号右边的表达式或者变量可以分别缩写为RHS表达式或者RHS变量
    • LHS：赋值等号左边的表达式或者变量可以分别缩写为LHS表达式或者LHS变量

      1.1.2. 阻塞赋值

  • 在赋值时先计算RHS部分的值，此时赋值语句不允许任何别的Verilog语句的干扰，直到现行的赋值完成时刻，即把RHS赋值给LHS的时刻，才允许别的赋值语句的执行。
  • 可综合的阻塞赋值操作在RHS不能设定有延迟，零延迟也不允许，理论上来说，此赋值语句与后面的赋值语句只有概念上的先后没有实质上的延迟。
  • 阻塞赋值的概念是指在同一个always块中，后面的赋值语句从概念上是在前一赋值语句结束之后再开始赋值
  • 注意：如果在一个过程块中阻塞赋值的RHS变量正好是另一个过程块中阻塞赋值的LHS变量，这两个过程块又用同一个时钟沿触发，则可能出现竞争。若两个阻塞赋值操作用同一个时钟沿触发，则执行的顺序是无法确定的。
  • 实例

    1.1.3. 非阻塞赋值

  • 非阻塞赋值在赋值操作时刻开始时计算非阻塞赋值符的RHS表达式，赋值操作结束时刻更新LHS。在计算非阻塞赋值的RHS表达式和更新LHS期间，其他的verilog语句，包括其他的非阻塞赋值语句都能够同时运行。
  • 非阻塞赋值，当赋值操作所处的块语句结束时，赋值操作才会结束。
  • 非阻塞赋值操作过程可以看做两个步骤
    • 在赋值开始时刻，计算非阻塞赋值RHS表达式
    • 在赋值结束时刻，更新非阻塞赋值LHS表达式
  • 非阻塞赋值操作只能用于对寄存器类型变量进行赋值，只能用于always,initial等过程块中。同时不允许用于连续赋值。
  • 实例

    1.2. Verilog模块编程要点

  • 时序电路建模时，用非阻塞赋值。
  • 锁存器电路建模时，用非阻塞赋值。
  • 用always块建立组合逻辑模型时，用阻塞赋值。
  • 在同一个always块中建立时序和组合逻辑电路时，用非阻塞赋值。
  • 在同一个always块中不要既用非阻塞赋值又用阻塞赋值。
  • 不要在一个以上的always块中为同一个变量赋值
  • 用$strobe系统任务显示非阻塞赋值的变量值
  • 赋值时不要使用#0延迟

    1.3. Verilog层次化事件队列

  • 概念：层次化事件队列指的是用于调度仿真事件的不同的Verilog事件队列。在语言标准中被视为一个概念模型
  • 层次化事件队列在逻辑上分为用于当前仿真时间的4个不同的队列，和用于下一段仿真时间的若干个附加队列。
  • 层次化事件队列
    • 动态事件队列(下列事件执行顺序随意)
      • 阻塞赋值
      • 计算非阻塞赋值语句右边的表达式
      • 连续赋值
      • 执行$display命令
      • 计算原语的输入和输出的变化
    • 停止运行的事件队列：#0延时阻塞赋值
    • 非阻塞事件队列：更新非阻塞赋值语句LHS的值
    • 监控事件队列
      • 执行$monitor命令
      • 执行$strobe命令
    • 其他指定的PLI命令队列：其他PLI命令
  • 执行过程
    • 加入到上述队列中的事件只能从动态事件队列中清除，而排列在其他队列中的事件要等待被排入动态事件队列中后才能真正开始等待执行。

    1.9. 常见的对于非阻塞赋值的误解

  • 误解一：使用$display命令不能用来显示非阻塞赋值语句的赋值
    • 实际上非阻塞语句的赋值在所有的$display命令执行之后才会更新数值
  • 误解二：#0延时把赋值强制到仿真时间步的末尾
    • 实际上#0延时把赋值事件强制加入停止运行事件队列中。
  • 误解三：可以在同一个always块中对某一变量进行多次非阻塞赋值
    • 实际上，可以多次非阻塞赋值，但是只有最后一次的赋值起作用。

  • 概述：三种基本描述方式
    • 数据流描述：采用assign连续赋值语句
    • 行为描述：使用always语句或者initial语句中的过程赋值语句
    • 结构化描述：实例化已有的功能模块或者原语

      数据流描述

      数据流
  • 概念：信号在经过组合逻辑时从输入流向输出，并不会在其中存储。类似于数据流动。
    连续赋值语句
  • 实例：assign #1 A = a^b;
  • 特点
    • 连续驱动：只要输入发生变化，就会导致该语句重新计算。
    • 只有线网类型的变量才能在assign语句中赋值.
      • 因为连续赋值语句中被赋值的变量在仿真器中不会存储其值，因此只能是线网类型而不能是寄存器类型。
      • 线网类型的变量可以被多重驱动，即可以在多个连续赋值语句中驱动同一个线网
    • 使用assign对组合逻辑建模
      • 建议使用数据流描述对组合逻辑进行建模。因为assign语句的连续驱动特点与组合逻辑非常相似。
    • 并行性
      • assign语句与行为语句块always,initial等、其他连续赋值语句、门级模型之间是并行的。一个连续赋值语句就是一个独立的进程，进程之间是并发的。
        延时
  • 在连续赋值语句中可以对电路的延时进行建模。如上述例子中的延时表示该异或门的延时为1个时间单位，即为从输入端信号变化到输出端体现需要1个时间单位。
  • assign语句的延时特性通常被逻辑综合工具忽略。
    多驱动源线网
  • 多重驱动wire，错误用法
  • 线与、线或功能
    • 可以使用wor线网类型将不同的输出线或在一起，线与使用wand
  • 三态总线功能
    • tri线网类型

      行为描述

  • 概念：即为用语言描述电路的行为。有initial语句和always语句
  • 注意：程序运行时，initial和always语句同时执行，但是执行顺序随机。
    过程块中的语句种类

    • 非阻塞过程赋值
    • 阻塞过程赋值

      设计原则

      面积与速度的平衡与互换原则
  • 基本概念
    • 面积：一个设计所消耗的目标器件的硬件资源数量。
    • 速度：设计在芯片上稳定运行时所能达到的最高频率。
  • 基本关系
    • 二者是一对对立统一的矛盾体，是衡量设计质量的终极标准。二者同时满足是不现实的。二者在设计中的地位是不同的。满足时序、工作频率的要求更重要，速度优先。所以设计的目标应该是在满足时序要求的前提下，占用最小的芯片面积。
  • 基本思想
    • 速度换面积：一个设计如果时序余量较大，所能实现的时序频率远高于设计要求，此时可以通过功能块复用减少整个设计所消耗的面积
    • 面积换速度：如果一个设计所要求的时序很高，普通方法无法达到。一般可通过将数据流串并转换，并赋值多个操作模块的方式提高频率。
