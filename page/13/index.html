常用伪指令

概述

  • 伪指令用于告知汇编器根据指定的要求对汇编程序进行汇编。
  • 使用伪指令可以简化编程过程，是程序更加规范、易读、易交流。
  • ARM中的伪指令和X86中的伪指令很相似但有不同。ARM中的有些伪指令可以汇编成一条至多条机器指令，而X86的伪指令无法汇编为真正的机器指令。

常用指令

跳转指令

  • B [cond] label

    • Branch

    • 跳转到lable指向的位置

    • 采用相对寻址，PC = PC + label，跳转范围-32M ~ +32M。

    • 共24位表示偏移量，但是偏移量后两位一定为0，所以实际为26位。

  • BL [cond] label

    • Branch and Link
    • 跳转的同时保存断点，常用于子程序调用。
    • 采用相对寻址.LR = PC - 4 ,PC = PC + label.此时LR存储本跳转指令的下一条指令地址,作为程序返回之后的执行起点.
    • BL机器指令格式同上

  • BX [cond] Rm

    • Branch and Exchange
    • 跳转的同时切换处理器状态
    • 如果Rm的0位为1,则将CPSR的T标志位置位,切换到Thumb模式;否则T标志位复位,切换到ARM模式
    • 采用相对寻址,PC = PC + Rm

  • BLX [cond] label|Rm

    • 跳转的同时保存断点并可进行模式切换

存储器访问指令

共有三种类型的存储器访问指令

  • 单寄存器传输指令,LDR,STR
  • 多寄存器传输指令,LDM,STM
  • 交换指令,SWP
LDR、STR指令

  • Load Register`storage Register`

  • 从内存地址中加载一个字到寄存器.

  • 语法

    语法及指令格式
    LDR\STR[cond] Rd,addressing_mod
    • Rd:目标寄存器,将取出的数据存入此寄存器
    • addressing_mod:
指令名称 操作 指令描述
LDR 将一个字装入寄存器 LDR Rn,基址变址寻址,将地址中存储一个字放入Rn寄存器
STR 将寄存器中的值存入内存单元 STR Rn,基址变址寻址,将Rn中的内容存入给定的地址中
LDRB/STRB 将一个字节装入寄存器/ LDRB Rn,基址变址寻址,将地址中的存储的一个字节放入Rn寄存器
STRH/STRH 将一个半字装入寄存器/
STM、LDM指令

  • Store Multiple，Load Multiple

  • 将多个寄存器内容存储到指定内存

  • 会增加中断的延迟，因为ARM不会打断正在执行的指令，而去响应中断。一般编译器可以通过设置一次传送的最大寄存器数目来限制此延迟。

  • 语法

    语法及机器指令格式
    STM/LDM[cond]<addressing_mode> <Rn>[!],<register>[^]

    • cond:决定指令是否执行的条件

    • addressing_mode:

      • IA:increment after,传送后地址加4.操作堆栈时等于EA,空递增堆栈
      • IB:increment before,传送前地址加4.操作堆栈时等于FA,满递增堆栈
      • DA:decrement after,传送后地址减4.操作堆栈时等于ED,空递减堆栈
      • DB:Decrement before,传送前地址减4.操作堆栈时等于FD,满递减堆栈

    • Rn:Rn的值作为操作的基地址

    • !:操作完成之后将新的地址写回Rn

    • register:用{}括起来的寄存器列表。寄存器之间用,隔开,连续的寄存器可用-简写。

      小号寄存器存入低地址,大号存储器存入高地址。register_list16位对应16个寄存器,相应位为1,则代表选中相应的寄存器。

    • ^:将当前模式下的SPSR写入CPSR

  • 注意

    • 在指令中有!,且Rn在寄存器列表中时,如果Rn不是其中编号最小的寄存器,则会发生错误,结果不可预测.
SWP指令

  • signal data swap，单数据交换指令

  • 是原子操作，可用于实现信号量和互斥

  • 语法

    语法及机器指令格式
    SWP{cond}{B}Rd,Rm,[Rn]
    • cond：决定指令是否执行的条件
    • B：决定是交换字还是字节
    • Rd、Rm、[Rn]：将[Rn]地址对应的数据加载到Rd，并将Rm的值加载到[Rn]中

ARM指令系统

指令集语法规则

  • ARM汇编语言指令语法格式

    opcode [Cond] [S] Rd,[Rn],[Operand2]

  • 解释

    • opcode：4位，必选项。决定指令的操作。

    • Cond：4位，可选项。指令执行的条件，满足条件再执行。

      编码 助记符 标志 含义
      0000 EQ(equal) Z=1 相等
      0001 NE(not equal) Z=0 不相等
      0010 CS/HS(carry set/high or same) C=1 无符号数大于或等于
      0011 CC/LO(carry clear/lower) C=0 无符号数小于
      0100 MI(minus) N=1 负数
      0101 PL(plus) N=0 非负数
      0110 VS(overflow set) V=1 溢出
      0111 VC(overflow clear) V=0 未溢出
      1000 HI(high) C=1,Z=0 无符号数大于
      1001 LS(lower or same) C=0,Z=1 无符号数小于或等于
      1010 GE(greater or euqal) N=V 有符号数大于或等于
      1011 LT(less than) N != V 有符号数小于
      1100 GT(greater than) Z=0,N=V 有符号数大于
      1101 AL(always) 任何 无条件执行
      1111 NV(never) 任何 从不执行

    • S：1位，可选项。决定指令是否影响CPSR寄存器对应标志。

    • Rd：4位，必选项。目标寄存器。

    • Rn：4位，可选项。第一个操作数所在寄存器。

    • Operand2：12位，可选项。第二操作数。有两种形式

      • 立即数形式

        • 机器指令格式

          #rotate #immed_8r

          内容 含义
          #rotate(11:8) 表示循环右移的次数的$\frac{1}{2}$
          #immed_8r(7:0) 表示需要循环右移的数

        • 实际书写格式:

          #满足条件的立即数

        • 最终结果为#immed_8r循环右移2*#rotote所得的值.所以赋值时必须给一个可以通过8位立即数循环右移偶数次得到的立即数,才能正确的表示.

      • 寄存器移位型

        • 将寄存器的移位结果作为操作数

        • 指令格式

          [Rm [,mode #imm]]

          内容 含义
          Rm(3:0) 可省略，但是有shift时不能省略。存储原始数据的寄存器
          mode(6:4) 移位操作符。指定移位规则
          #imm(11:7) 移位次数。小于32的立即数

        • 移位模式

          模式 描述
          LSL(logic shift keft) 逻辑左移/算数左移，补0
          LSR(logic shift right) 逻辑右移，补0
          ASR 算数右移，保持符号位不变，空出位置补符号位
          ROR 循环右移，低端移除填入高端
          RRX 带进位的循环右移1位，后不跟移位次数。第31位用原来的C填入，如果指定后缀S，则将原0位填入C中

          没有专门的算数左移指令的原因，来源

  • ARM机器指令语法格式

寻址方式

立即寻址

  • 操作数本身写在指令当中。

  • 立即数只能在第二操作数中给出，需满足约束条件。

  • 一般以#作为前缀，#0x十六进制、#0d十进制、#0b二进制。

  • 实例

寄存器寻址、寄存器移位寻址
  • 第二操作数采用第二种形式的寻址方式。
  • 将寄存器的移位结果作为操作数。省略整个第二操作数、或者省略移位、或者使用完整形式。
  • 只有寄存器时为寄存器寻址，有移位时为寄存器移位寻址。
基址变址寻址、寄存器间接寻址

  • 前变址

    • 首先计算Rn+Operand2得到地址,然后完成指令操作.
    • Operand2可以取第二操作数的任何形式.

  • 带写回的前变址

    • 首先计算Rn+Operand2得到地址,然后完成指令操作,最后将地址写回到Rn寄存器
    • Operand2可以取第二操作数的任何形式.

  • 后变址

    • 根据Rn得到地址,然后完成指令操作,最后将Rn+Operand2的值写回到Rn寄存器
    • Operand2可以取第二操作数的任何形式

01.程序的组成

汇编语言由定义好的段组成，每个段都有不同的目的。三个最常用的段如下：

  • data段，是可选的，声明赋有初始值的数据元素，这些数据元素常用做汇编语言程序中的变量。
  • bss段，是可选的，声明没有赋初始值的数据元素（默认使用0初始化），这些数据元素常用做汇编语言程序中的缓冲区。
  • text段，所有汇编程序必须有文本段，文本段是声明指令代码的地方。

数据依赖与关系模式规范化

规范化算法

即无损连接又保持函数依赖的3NF分解算法
  • 输入：关系模式$R_R(U_R,F_R)$
  • 输出：$\rho={R_0,R_1,\dots,R_k,R_{ck}}$
  • 步骤
    • 将$F_R$转换为等价的最小覆盖，记为$F$
    • 求关系模式的候选键,记作$U_{ck}^,$
    • 将不在$F$中出现的属性单独构成关系模式$R_0(U_0)$.其余属性记为$U=U_R-U_0$,构成关系$R(U,F)$.于是得到一个分解$\rho={R_0,R}$
    • 如果$R$中有一个函数依赖包含$U$中的全部属性,则终止,输出$\rho$
    • 否则,将$F$中的函数依赖按照决定子相同的原则进行分组.假设得到k个组,则构成k个关系模式$R_i(U_i,F_i)$,$U_i$表示每组函数依赖涉及的全部属性,$F_i$为相应的函数依赖集,暂不计算.如果存在$U_i\subseteq U_j$,则略去$U_i$.用这k个模式替换原来的$R$,得到$\rho={R_0,R_1,\dots,R_k}$
    • 最后,再加入一个由任意一个候选键构成的关系模式$R_{ck}(U_{ck},F_{ck})$,$U_{ck}$为任一个候选键,$F_{ck}$为相应的函数依赖集,暂不计算.如果$U_{ck}\subseteq U_i$,则略去.此时得到$\rho={R_0,R_1,\dots,R_k,R_{ck}}$
    • 每个关系的函数依赖集通过$F$在$U_i$上的投影算出.
    • 输出$\rho$,算法结束
保证无损连接的BCNF分解算法
  • 输入:关系模式$R(U,F)$
  • 输出:$\rho={R_1,\dots,R_k}$
  • 步骤
    • 令$\rho={R(U,F)}$
    • 如果$\rho$中的所有子关系模式都是BCNF的,则算法终止,输出$\rho$
    • 否则,任取$\rho$中一个不满足BCNF的关系模式$R_i(U_i,F_i)$。根据BCNF的定义,至少有一个$X\rightarrow A\in F_i^+$,且X不包含$R_i$的任一候选键。将$R_i$分解为两个关系模式$R_{i1}$、$R_{i2}$,且$R_{i1}={XA,F_{i1}}$,$R_{i2}={U_i-A,F_{i2}}$.用$R_{i1},R_{i2}$替换$R_i$,返回上一步继续执行。相应的函数依赖通过F在属性集合上的投影算出。
